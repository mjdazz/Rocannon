SYNOPSIS                   *fortios_router_key_chain*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify router feature and key_chain category. Examples include all parameters and values
       need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_router_key_chain-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_router_key_chain-parameters*

        Parameter      Choices/Defaults                                                                           Comments
   host                                 FortiOS or FortiGate IP address.
   string
   https                 Choices:
   boolean               • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                         • yes ←
   password            Default:         FortiOS or FortiGate password.
   string              ""
   router_key_chain    Default:         Configure key-chain.
   dictionary          null
    key                                 Configuration method to edit key settings.
    list
     accept_lifetime                    Lifetime of received authentication key (format: hh:mm:ss day month year).
     string
     id                                 Key ID (0 - 2147483647).
     string / required
     key_string                         Password for the key (max. = 35 characters).
     string
     send_lifetime                      Lifetime of sent authentication key (format: hh:mm:ss day month year).
     string
    name                                Key-chain name.
    string / required
   ssl_verify            Choices:
   boolean               • no           Ensures FortiGate certificate must be verified by a proper CA.
                         • yes ←
   state                 Choices:
   string / required     • present      Indicates whether to create or remove the object.
                         • absent
   username                             FortiOS or FortiGate username.
   string
   vdom                Default:         Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different unit.
   string              "root"

NOTES                   *fortios_router_key_chain-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_router_key_chain-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure key-chain.
     fortios_router_key_chain:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       state: "present"
       router_key_chain:
         key:
          -
             accept_lifetime: "<your_own_value>"
             id:  "5"
             key_string: "<your_own_value>"
             send_lifetime: "<your_own_value>"
         name: "default_name_8"
