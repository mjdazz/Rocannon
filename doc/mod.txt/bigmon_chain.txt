SYNOPSIS                   *bigmon_chain*

     • Create and remove a bigmon inline service chain.

PARAMETERS                   *bigmon_chain-parameters*

     Parameter    Choices/Defaults                                                                              Comments
   access_token                    Bigmon access token. If this isn't set, the environment variable BIGSWITCH_ACCESS_TOKEN is used.
   -
   controller                      The controller IP address.
   - / required
   name                            The name of the chain.
   - / required
   state            Choices:
   -                • present ←    Whether the service chain should be present or absent.
                    • absent
   validate_certs   Choices:
   boolean          • no           If false, SSL certificates will not be validated. This should only be used on personally controlled devices using self-signed certificates.
                    • yes ←

EXAMPLES                   *bigmon_chain-examples*

 - name: bigmon inline service chain
   bigmon_chain:
     name: MyChain
     controller: '{{ inventory_hostname }}'
     state: present
     validate_certs: false
