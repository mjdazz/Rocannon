SYNOPSIS                   *hipchat*

     • Send a message to a Hipchat room, with options to control the formatting.

PARAMETERS                   *hipchat-parameters*

     Parameter          Choices/Defaults                                                                              Comments
   api            Default:                     API url if using a self-hosted hipchat server. For Hipchat API version 2 use the default URI with /v2 instead of /v1.
   -              "https://api.hipchat.com/v1"
                    Choices:
                    • yellow ←
   color            • red
   -                • green                    Background color for the message.
                    • purple
                    • gray
                    • random
   from           Default:                     Name the message will appear to be sent from. Max length is 15 characters - above this it will be truncated.
   -              "Ansible"
   msg                                         The message body.
   - / required
   msg_format       Choices:
   -                • text ←                   Message format.
                    • html
   notify           Choices:
   boolean          • no                       If true, a notification will be triggered for users in the room.
                    • yes ←
   room                                        ID or name of the room.
   - / required
   token                                       API token.
   - / required
   validate_certs   Choices:
   boolean          • no                       If no, SSL certificates will not be validated. This should only be used on personally controlled sites using self-signed certificates.
                    • yes ←

EXAMPLES                   *hipchat-examples*

 - hipchat:
     room: notif
     msg: Ansible task finished

 # Use Hipchat API version 2
 - hipchat:
     api: https://api.hipchat.com/v2/
     token: OAUTH2_TOKEN
     room: notify
     msg: Ansible task finished
