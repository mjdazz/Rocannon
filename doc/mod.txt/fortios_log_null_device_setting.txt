SYNOPSIS                   *fortios_log_null_device_setting*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify log_null_device feature and setting category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_log_null_device_setting-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_log_null_device_setting-parameters*

          Parameter        Choices/Defaults                                                                         Comments
   host                                     FortiOS or FortiGate IP address.
   string
   https                     Choices:
   boolean                   • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                             • yes ←
   log_null_device_setting Default:         Settings for null device logging.
   dictionary              null
            status           Choices:
            string           • enable       Enable/disable statistics collection for when no external logging destination, such as FortiAnalyzer, is present (data is not saved).
                             • disable
   password                Default:         FortiOS or FortiGate password.
   string                  ""
   ssl_verify                Choices:
   boolean                   • no           Ensures FortiGate certificate must be verified by a proper CA.
   added in 2.9              • yes ←
   username                                 FortiOS or FortiGate username.
   string
   vdom                    Default:         Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different unit.
   string                  "root"

NOTES                   *fortios_log_null_device_setting-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_log_null_device_setting-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Settings for null device logging.
     fortios_log_null_device_setting:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       log_null_device_setting:
         status: "enable"
