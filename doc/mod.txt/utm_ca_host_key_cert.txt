SYNOPSIS                   *utm_ca_host_key_cert*

     • Create, update or destroy a ca host_key_cert entry in SOPHOS UTM.
     • This module needs to have the REST Ability of the UTM to be activated.

PARAMETERS                   *utm_ca_host_key_cert-parameters*

     Parameter    Choices/Defaults                                                                              Comments
   ca                              A reference to an existing utm_ca_signing_ca or utm_ca_verification_ca object.
   - / required
   certificate                     The certificate in PEM format.
   - / required
   comment                         Optional comment string.
   -
   encrypted        Choices:
   boolean          • no ←         Optionally enable encryption.
                    • yes
   headers                         A dictionary of additional headers to be sent to POST and PUT requests.
   dictionary                      Is needed for some modules
   key                             Optional private key in PEM format.
   -
   meta                            A reference to an existing utm_ca_meta_x509 object.
   - / required
   name                            The name of the object. Will be used to identify the entry.
   - / required
   state            Choices:       The desired state of the object.
   string           • absent       present will create or update an object
                    • present ←    absent will delete an object if it was present
   utm_host
   string /                        The REST Endpoint of the Sophos UTM.
   required
   utm_port       Default:         The port of the REST interface.
   integer        4444
   utm_protocol     Choices:
   string           • http         The protocol of the REST Endpoint.
                    • https ←
   utm_token                       The token used to identify at the REST-API. See https://www.sophos.com/en-us/medialibrary/PDFs/documentation/UTMonAWS/Sophos-UTM-RESTful-API.pdf?la=en, Chapter
   string /                        2.4.2.
   required
   validate_certs   Choices:
   boolean          • no           Whether the REST interface's ssl certificate should be verified or not.
                    • yes ←

EXAMPLES                   *utm_ca_host_key_cert-examples*

 # Create a ca_host_key_cert entry
 - name: utm ca_host_key_cert
   utm_ca_host_key_cert:
     utm_host: sophos.host.name
     utm_token: abcdefghijklmno1234
     name: TestHostKeyCertEntry
     ca: REF_ca/signing_ca_OBJECT_STRING
     meta: REF_ca/meta_x509_OBJECT_STRING
     certificate: |
       --- BEGIN CERTIFICATE ---
       . . .
        . . .
       . . .
       --- END CERTIFICATE ---
     state: present

 # Remove a ca_host_key_cert entry
 - name: utm ca_host_key_cert
   utm_ca_host_key_cert:
     utm_host: sophos.host.name
     utm_token: abcdefghijklmno1234
     name: TestHostKeyCertEntry
     state: absent

 # Read a ca_host_key_cert entry
 - name: utm ca_host_key_cert
   utm_ca_host_key_cert:
     utm_host: sophos.host.name
     utm_token: abcdefghijklmno1234
     name: TestHostKeyCertEntry
     state: info
