SYNOPSIS                   *netapp_e_snapshot_volume*

     • Create, update, remove snapshot volumes for NetApp E/EF-Series storage arrays.

PARAMETERS                   *netapp_e_snapshot_volume-parameters*

       Parameter     Choices/Defaults                                                                             Comments
   api_password                        The password to authenticate with the SANtricity WebServices Proxy or embedded REST API.
   - / required
   api_url                             The url to the SANtricity WebServices Proxy or embedded REST API.
   - / required
   api_username                        The username to authenticate with the SANtricity WebServices Proxy or embedded REST API.
   - / required
   full_threshold    Default:          The repository utilization warning threshold percentage
   -                 85
   name                                The name you wish to give the snapshot volume
   - / required
   repo_percentage   Default:          The size of the view in relation to the size of the base volume
   -                 20
   snapshot_image_id                   The identifier of the snapshot image used to create the new snapshot volume.
   - / required                        Note: You'll likely want to use the netapp_e_facts module to find the ID of the image you want.
   ssid                                storage array ID
   - / required
   state               Choices:
   - / required        • absent        Whether to create or remove the snapshot volume
                       • present
   storage_pool_name                   Name of the storage pool on which to allocate the repository volume.
   - / required
   validate_certs      Choices:
   boolean             • no            Should https certificates be validated?
                       • yes ←
                       Choices:
   view_mode           • modeUnknown
   - / required        • readWrite     The snapshot volume access mode
                       • readOnly
                       • __UNDEFINED

NOTES                   *netapp_e_snapshot_volume-notes*

   Note

     • Only full_threshold is supported for update operations. If the snapshot volume already exists and the threshold matches, then an ok status will be returned, no other changes can be made to a
       pre-existing snapshot volume.

EXAMPLES                   *netapp_e_snapshot_volume-examples*

 - name: Snapshot volume
   netapp_e_snapshot_volume:
     ssid: "{{ ssid }}"
     api_url: "{{ netapp_api_url }}/"
     api_username: "{{ netapp_api_username }}"
     api_password: "{{ netapp_api_password }}"
     state: present
     storage_pool_name: "{{ snapshot_volume_storage_pool_name }}"
     snapshot_image_id: "{{ snapshot_volume_image_id }}"
     name: "{{ snapshot_volume_name }}"
