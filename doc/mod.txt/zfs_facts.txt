SYNOPSIS                   *zfs_facts*

     • Gather facts from ZFS dataset properties.

PARAMETERS                   *zfs_facts-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   depth                         Specifies recursion depth.
   -
   name                          ZFS dataset name.
   - / required                  aliases: ds, dataset
   parsable       Choices:
   boolean        • no ←         Specifies if property values should be displayed in machine friendly format.
                  • yes
   properties   Default:         Specifies which dataset properties should be queried in comma-separated format. For more information about dataset properties, check zfs(1M) man page.
   -            "all"            aliases: props
   recurse        Choices:
   boolean        • no ←         Specifies if properties for any children should be recursively displayed.
                  • yes
                  Choices:
                  • all ←
   type           • filesystem   Specifies which datasets types to display. Multiple values have to be provided in comma-separated form.
   -              • volume
                  • snapshot
                  • bookmark

EXAMPLES                   *zfs_facts-examples*

 - name: Gather facts about ZFS dataset rpool/export/home
   zfs_facts:
     dataset: rpool/export/home

 - name: Report space usage on ZFS filesystems under data/home
   zfs_facts:
     name: data/home
     recurse: yes
     type: filesystem

 - debug:
     msg: 'ZFS dataset {{ item.name }} consumes {{ item.used }} of disk space.'
   with_items: '{{ ansible_zfs_datasets }}'
