SYNOPSIS                   *pn_admin_session_timeout*

     • This module can be used to modify admin session timeout.

PARAMETERS                   *pn_admin_session_timeout-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   pn_cliswitch                       Target switch to run the CLI on.
   string
   pn_timeout                         Maximum time to wait for user activity before terminating login session. Minimum should be 60s.
   string
   state               Choices:       State the action to perform. update to modify the admin-session-timeout.
   string / required   • update

EXAMPLES                   *pn_admin_session_timeout-examples*

 - name: admin session timeout functionality
   pn_admin_session_timeout:
     pn_cliswitch: "sw01"
     state: "update"
     pn_timeout: "61s"

 - name: admin session timeout functionality
   pn_admin_session_timeout:
     pn_cliswitch: "sw01"
     state: "update"
     pn_timeout: "1d"

 - name: admin session timeout functionality
   pn_admin_session_timeout:
     pn_cliswitch: "sw01"
     state: "update"
     pn_timeout: "10d20m3h15s"

RETURN VALUES                   *pn_admin_session_timeout-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   indicates whether the CLI caused changes on the target.
   boolean
   command always   the CLI command run on the target node.
   string
   stderr  on error set of error responses from the admin-session-timeout command.
   list
   stdout  always   set of responses from the admin-session-timeout command.
   list
