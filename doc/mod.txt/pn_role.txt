SYNOPSIS                   *pn_role*

     • This module can be used to create, delete and modify user roles.

PARAMETERS                   *pn_role-parameters*

        Parameter       Choices/Defaults                                                                           Comments
   pn_access              Choices:
   string                 • read-only    type of access.
                          • read-write
   pn_cliswitch                          Target switch to run the CLI on.
   string
   pn_delete_from_users   Choices:
   boolean                • no           delete from users.
                          • yes
   pn_name                               role name.
   string / required
   pn_running_config      Choices:
   boolean                • no           display running configuration of switch.
                          • yes
   pn_scope               Choices:
   string                 • local        local or fabric.
                          • fabric
   pn_shell               Choices:
   boolean                • no           allow shell command.
                          • yes
   pn_sudo                Choices:
   boolean                • no           allow sudo from shell.
                          • yes
                          Choices:
   state                  • present      State the action to perform. Use present to create role and absent to delete role and update to modify role.
   string / required      • absent
                          • update

EXAMPLES                   *pn_role-examples*

 - name: Role create
   pn_role:
     pn_cliswitch: 'sw01'
     state: 'present'
     pn_name: 'foo'
     pn_scope: 'local'
     pn_access: 'read-only'

 - name: Role delete
   pn_role:
     pn_cliswitch: 'sw01'
     state: 'absent'
     pn_name: 'foo'

 - name: Role modify
   pn_role:
     pn_cliswitch: 'sw01'
     state: 'update'
     pn_name: 'foo'
     pn_access: 'read-write'
     pn_sudo: true
     pn_shell: true

RETURN VALUES                   *pn_role-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   indicates whether the CLI caused changes on the target.
   boolean
   command always   the CLI command run on the target node.
   string
   stderr  on error set of error responses from the role command.
   list
   stdout  always   set of responses from the role command.
   list
