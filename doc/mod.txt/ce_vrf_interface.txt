SYNOPSIS                   *ce_vrf_interface*

     • Manages interface specific VPN configuration of HUAWEI CloudEngine switches.

PARAMETERS                   *ce_vrf_interface-parameters*

     Parameter   Choices/Defaults                                                                              Comments
   state           Choices:
   -               • present ←    Manage the state of the resource.
                   • absent
   vpn_interface                  An interface that can binding VPN instance, i.e. 40GE1/0/22, Vlanif10. Must be fully qualified interface name. Interface types, such as 10GE, 40GE, 100GE,
   - / required                   LoopBack, MEth, Tunnel, Vlanif....
   vrf                            VPN instance, the length of vrf name is 1 ~ 31, i.e. "test", but can not be _public_.
   - / required

NOTES                   *ce_vrf_interface-notes*

   Note

     • Ensure that a VPN instance has been created and the IPv4 address family has been enabled for the VPN instance.
     • This module requires the netconf system service be enabled on the remote device being managed.
     • Recommended connection is netconf.
     • This module also works with local connections for legacy playbooks.

EXAMPLES                   *ce_vrf_interface-examples*

 - name: VRF interface test
   hosts: cloudengine
   connection: local
   gather_facts: no
   vars:
     cli:
       host: "{{ inventory_hostname }}"
       port: "{{ ansible_ssh_port }}"
       username: "{{ username }}"
       password: "{{ password }}"
       transport: cli

   tasks:

   - name: "Configure a VPN instance for the interface"
     ce_vrf_interface:
       vpn_interface: 40GE1/0/2
       vrf: test
       state: present
       provider: "{{ cli }}"

   - name: "Disable the association between a VPN instance and an interface"
     ce_vrf_interface:
       vpn_interface: 40GE1/0/2
       vrf: test
       state: absent
       provider: "{{ cli }}"

RETURN VALUES                   *ce_vrf_interface-return values*

   Common return values are documented here, the following are the fields unique to this module:

      Key       Returned                                                                                  Description
   changed                 check to see if a change was made on the device
   boolean    always       Sample:
                           True
   end_state               k/v pairs of end attributes on the interface
   dictionary verbose mode Sample:
                           {'vpn_interface': '40GE2/0/17', 'vrf': 'jss'}
   existing                k/v pairs of existing attributes on the interface
   dictionary verbose mode Sample:
                           {'vpn_interface': '40GE2/0/17', 'vrf': None}
   proposed                k/v pairs of parameters passed into module
   dictionary verbose mode Sample:
                           {'state': 'present', 'vpn_interface': '40GE2/0/17', 'vrf': 'jss'}
   updates                 command list sent to the device
   list       always       Sample:
                           ['ip binding vpn-instance jss']
