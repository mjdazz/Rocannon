SYNOPSIS                   *hcloud_volume_info*

     • Gather infos about your Hetzner Cloud volumes.

   Aliases: hcloud_volume_facts

REQUIREMENTS                   *hcloud_volume_info-requirements*

   The below requirements are needed on the host that executes this module.

     • hcloud-python >= 1.0.0

PARAMETERS                   *hcloud_volume_info-parameters*

       Parameter            Choices/Defaults                                                                            Comments
   api_token                                        This is the API Token for the Hetzner Cloud.
   string / required
   endpoint          Default:                       This is the API Endpoint for the Hetzner Cloud.
   string            "https://api.hetzner.cloud/v1"
   id                                               The ID of the volume you want to get.
   integer
   label_selector                                   The label selector for the volume you want to get.
   string
   name                                             The name of the volume you want to get.
   string

SEE ALSO                   *hcloud_volume_info-see also*

   See also

   Documentation for Hetzner Cloud API
   Complete reference for the Hetzner Cloud API.

EXAMPLES                   *hcloud_volume_info-examples*

 - name: Gather hcloud volume infos
   hcloud_volume_info:
   register: output
 - name: Print the gathered infos
   debug:
     var: output.hcloud_volume_info

RETURN VALUES                   *hcloud_volume_info-return values*

   Common return values are documented here, the following are the fields unique to this module:

          Key         Returned                                                                              Description
   hcloud_volume_info always   The volume infos as list
   complex
        id                     Numeric identifier of the volume
        integer       always   Sample:
                               1937415
        labels        always   User-defined labels (key-value pairs)
        dictionary
        location               Name of the location where the volume resides in
        string        always   Sample:
                               fsn1
        name                   Name of the volume
        string        always   Sample:
                               my-volume
        server                 Name of the server where the volume is attached to
        string        always   Sample:
                               my-server
        size                   Size of the volume
        string        always   Sample:
                               10
