SYNOPSIS                   *ejabberd_user*

     • This module provides user management for ejabberd servers

REQUIREMENTS                   *ejabberd_user-requirements*

   The below requirements are needed on the host that executes this module.

     • ejabberd with mod_admin_extra

PARAMETERS                   *ejabberd_user-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   host                          the ejabberd host associated with this username
   - / required
   logging        Choices:
   boolean        • no ←         enables or disables the local syslog facility for this module
                  • yes
   password                      the password to assign to the username
   -
   state          Choices:
   -              • present ←    describe the desired state of the user to be managed
                  • absent
   username                      the name of the user to manage
   - / required

NOTES                   *ejabberd_user-notes*

   Note

     • Password parameter is required for state == present only
     • Passwords must be stored in clear text for this release
     • The ejabberd configuration file must include mod_admin_extra as a module.

EXAMPLES                   *ejabberd_user-examples*

 # Example playbook entries using the ejabberd_user module to manage users state.

 - name: create a user if it does not exist
   ejabberd_user:
     username: test
     host: server
     password: password

 - name: delete a user if it exists
   ejabberd_user:
     username: test
     host: server
     state: absent
