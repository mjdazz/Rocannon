SYNOPSIS                   *cs_facts*

     • This module fetches data from the metadata API in CloudStack. The module must be called from within the instance itself.

REQUIREMENTS                   *cs_facts-requirements*

   The below requirements are needed on the host that executes this module.

     • yaml

PARAMETERS                   *cs_facts-parameters*

     Parameter             Choices/Defaults                                                                              Comments
                    Choices:
                    • cloudstack_service_offering
                    • cloudstack_availability_zone
   filter           • cloudstack_public_hostname
   string           • cloudstack_public_ipv4         Filter for a specific fact.
                    • cloudstack_local_hostname
                    • cloudstack_local_ipv4
                    • cloudstack_instance_id
                    • cloudstack_user_data
   meta_data_host                                    Host or IP of the meta data API service.
   string                                            If not set, determination by parsing the dhcp lease file.
   added in 2.4

EXAMPLES                   *cs_facts-examples*

 # Gather all facts on instances
 - name: Gather cloudstack facts
   cs_facts:

 # Gather specific fact on instances
 - name: Gather cloudstack facts
   cs_facts: filter=cloudstack_instance_id

 # Gather specific fact on instances with a given meta_data_host
 - name: Gather cloudstack facts
   cs_facts:
     filter: cloudstack_instance_id
     meta_data_host: 169.254.169.254

RETURN VALUES                   *cs_facts-return values*

   Common return values are documented here, the following are the fields unique to this module:

               Key              Returned                                                                         Description
   cloudstack_availability_zone          zone the instance is deployed in.
   string                       success  Sample:
                                         ch-gva-2
   cloudstack_instance_id                UUID of the instance.
   string                       success  Sample:
                                         ab4e80b0-3e7e-4936-bdc5-e334ba5b0139
   cloudstack_local_hostname             local hostname of the instance.
   string                       success  Sample:
                                         VM-ab4e80b0-3e7e-4936-bdc5-e334ba5b0139
   cloudstack_local_ipv4                 local IPv4 of the instance.
   string                       success  Sample:
                                         185.19.28.35
   cloudstack_public_hostname            public IPv4 of the router. Same as cloudstack_public_ipv4.
   string                       success  Sample:
                                         VM-ab4e80b0-3e7e-4936-bdc5-e334ba5b0139
   cloudstack_public_ipv4                public IPv4 of the router.
   string                       success  Sample:
                                         185.19.28.35
   cloudstack_service_offering           service offering of the instance.
   string                       success  Sample:
                                         Micro 512mb 1cpu
   cloudstack_user_data                  data of the instance provided by users.
   dictionary                   success  Sample:
                                         {'bla': 'foo'}
