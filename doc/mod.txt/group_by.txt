SYNOPSIS                   *group_by*

     • Use facts to create ad-hoc groups that can be used later in a playbook.
     • This module is also supported for Windows targets.

PARAMETERS                   *group_by-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   key                                The variables whose values will be used as groups.
   string / required
   parents           Default:
   list              "all"            The list of the parent groups.
   added in 2.4

NOTES                   *group_by-notes*

   Note

     • Spaces in group names are converted to dashes ‘-‘.
     • This module is also supported for Windows targets.

SEE ALSO                   *group_by-see also*

   See also

   add_host – Add a host (and alternatively a group) to the ansible-playbook in-memory inventory
   The official documentation on the add_host module.

EXAMPLES                   *group_by-examples*

 # Create groups based on the machine architecture
 - group_by:
     key: machine_{{ ansible_machine }}

 # Create groups like 'virt_kvm_host'
 - group_by:
     key: virt_{{ ansible_virtualization_type }}_{{ ansible_virtualization_role }}

 # Create nested groups
 - group_by:
     key: el{{ ansible_distribution_major_version }}-{{ ansible_architecture }}
     parents:
       - el{{ ansible_distribution_major_version }}
