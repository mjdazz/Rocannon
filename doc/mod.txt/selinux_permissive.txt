SYNOPSIS                   *selinux_permissive*

     • Add and remove a domain from the list of permissive domains.

REQUIREMENTS                   *selinux_permissive-requirements*

   The below requirements are needed on the host that executes this module.

     • policycoreutils-python

PARAMETERS                   *selinux_permissive-parameters*

       Parameter      Choices/Defaults                                                                            Comments
   domain             Default:         The domain that will be added or removed from the list of permissive domains.
   string / required  ""               aliases: name
   no_reload            Choices:       Disable reloading of the SELinux policy after making change to a domain's permissive setting.
   boolean              • no ←         The default is no, which causes policy to be reloaded when a domain changes state.
                        • yes          Reloading the policy does not work on older versions of the policycoreutils-python library, for example in EL 6."
   permissive           Choices:
   boolean / required   • no           Indicate if the domain should or should not be set as permissive.
                        • yes
   store                               Name of the SELinux policy store to use.
   string

NOTES                   *selinux_permissive-notes*

   Note

     • Requires a recent version of SELinux and policycoreutils-python (EL 6 or newer).

EXAMPLES                   *selinux_permissive-examples*

 - name: Change the httpd_t domain to permissive
   selinux_permissive:
     name: httpd_t
     permissive: true
