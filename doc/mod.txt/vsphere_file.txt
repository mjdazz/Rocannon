SYNOPSIS                   *vsphere_file*

     • Manage files on a vCenter datastore.

PARAMETERS                   *vsphere_file-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   datacenter                         The datacenter on the vCenter server that holds the datastore.
   string / required
   datastore                          The datastore on the vCenter server to push files to.
   string / required
   host                               The vCenter server on which the datastore is available.
   string / required                  aliases: hostname
   password                           The password to authenticate on the vCenter server.
   string / required
   path                               The file or directory on the datastore on the vCenter server.
   string / required                  aliases: dest
                       Choices:       The state of or the action on the provided path.
   state               • absent       If absent, the file will be removed.
   string              • directory    If directory, the directory will be created.
                       • file ←       If file, more information of the (existing) file will be returned.
                       • touch        If touch, an empty file will be created if the path does not exist.
   timeout           Default:         The timeout in seconds for the upload to the datastore.
   integer           10
   username                           The user name to authenticate on the vCenter server.
   string / required
   validate_certs      Choices:
   boolean             • no           If no, SSL certificates will not be validated. This should only be set to no when no other option exists.
                       • yes ←

NOTES                   *vsphere_file-notes*

   Note

     • The vSphere folder API does not allow to remove directory objects.

EXAMPLES                   *vsphere_file-examples*

 - name: Create an empty file on a datastore
   vsphere_file:
     host: '{{ vhost }}'
     username: '{{ vuser }}'
     password: '{{ vpass }}'
     datacenter: DC1 Someplace
     datastore: datastore1
     path: some/remote/file
     state: touch
   delegate_to: localhost

 - name: Create a directory on a datastore
   vsphere_file:
     host: '{{ vhost }}'
     username: '{{ vuser }}'
     password: '{{ vpass }}'
     src: /other/local/file
     datacenter: DC2 Someplace
     datastore: datastore2
     path: other/remote/file
     state: directory
   delegate_to: localhost

 - name: Query a file on a datastore
   vsphere_file:
     host: '{{ vhost }}'
     username: '{{ vuser }}'
     password: '{{ vpass }}'
     datacenter: DC1 Someplace
     datastore: datastore1
     path: some/remote/file
     state: touch
   delegate_to: localhost
   ignore_errors: yes

 - name: Delete a file on a datastore
   vsphere_file:
     host: '{{ vhost }}'
     username: '{{ vuser }}'
     password: '{{ vpass }}'
     datacenter: DC2 Someplace
     datastore: datastore2
     path: other/remote/file
     state: absent
   delegate_to: localhost
