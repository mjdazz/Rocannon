SYNOPSIS                   *atomic_image*

     • Manage the container images on the atomic host platform.
     • Allows to execute the commands specified by the RUN label in the container image when present.

REQUIREMENTS                   *atomic_image-requirements*

   The below requirements are needed on the host that executes this module.

     • atomic
     • python >= 2.6

PARAMETERS                   *atomic_image-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   backend        Choices:
   -              • docker       Define the backend where the image is pulled.
   added in 2.4   • ostree
   name                          Name of the container image.
   - / required
   started        Choices:
   boolean        • no           Start or Stop the container.
                  • yes ←
                  Choices:
   state          • absent       The state of the container image.
   -              • latest ←     The state latest will ensure container image is upgraded to the latest version and forcefully restart container, if running.
                  • present

NOTES                   *atomic_image-notes*

   Note

     • Host should support atomic command.

EXAMPLES                   *atomic_image-examples*

 - name: Execute the run command on rsyslog container image (atomic run rhel7/rsyslog)
   atomic_image:
     name: rhel7/rsyslog
     state: latest

 - name: Pull busybox to the OSTree backend
   atomic_image:
     name: busybox
     state: latest
     backend: ostree
