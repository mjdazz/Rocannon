DEPRECATED                   *panos_commit*

                                      Removed in Ansible:
                  version: 2.12
        Why:      Consolidating code base.
    Alternative:  Use https://galaxy.ansible.com/PaloAltoNetworks/paloaltonetworks instead.

SYNOPSIS                   *panos_commit-synopsis*

     • PanOS module that will commit firewall’s candidate configuration on
     • the device. The new configuration will become active immediately.

REQUIREMENTS                   *panos_commit-requirements*

   The below requirements are needed on the host that executes this module.

     • pan-python

PARAMETERS                   *panos_commit-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   commit_changes_by
   list                               Commit changes made by specified admin
   added in 2.8
   commit_vsys
   list                               Commit changes for specified VSYS
   added in 2.8
   description
   string                             Commit description/comment
   added in 2.8
   interval          Default:         interval for checking commit job
   -                 0.5
   ip_address                         IP address (or hostname) of PAN-OS device.
   - / required
   password                           Password for authentication. If the value is not specified in the task, the value of environment variable ANSIBLE_NET_PASSWORD will be used instead.
   - / required
   sync                Choices:
   boolean             • no           if commit should be synchronous
                       • yes ←
   timeout                            timeout for commit job
   -
   username          Default:         Username for authentication. If the value is not specified in the task, the value of environment variable ANSIBLE_NET_USERNAME will be used instead if defined.
   -                 "admin"          admin will be used if nothing above is defined.

EXAMPLES                   *panos_commit-examples*

 # Commit candidate config on 192.168.1.1 in sync mode
 - panos_commit:
     ip_address: "192.168.1.1"
     username: "admin"
     password: "admin"

RETURN VALUES                   *panos_commit-return values*

   Common return values are documented here, the following are the fields unique to this module:

         Key       Returned                                                                                Description
   panos_commit
   complex         always   Information about commit job.
   added in 2.8
     job_id                 Palo Alto job ID for the commit operation. Only returned if commit job is launched on device.
     string        always   Sample:
                            139
     status_code            Palo Alto API status code. Null if commit is successful.
     string        always   Sample:
                            19
     status_detail          Palo Alto API detailed status message.
     string        always   Sample:
                            Configuration committed successfully
     status_text            Palo Alto API status text.
     string        always   Sample:
                            success
