SYNOPSIS                   *fortios_firewall_ssh_setting*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify firewall_ssh feature and setting category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_firewall_ssh_setting-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_firewall_ssh_setting-parameters*

   firewall_ssh_setting   Default:         SSH proxy settings.

    host_trusted_checking   Choices:
    string                  • enable       Enable/disable host trusted checking.
                            • disable

    string
    hostkey_ecdsa256                       ECDSA nid256 certificate used by SSH proxy. Source firewall.ssh.local-key.name.
    string
    hostkey_ecdsa384                       ECDSA nid384 certificate used by SSH proxy. Source firewall.ssh.local-key.name.
    string
    hostkey_ecdsa521                       ECDSA nid384 certificate used by SSH proxy. Source firewall.ssh.local-key.name.
    string
    hostkey_ed25519                        ED25519 hostkey used by SSH proxy. Source firewall.ssh.local-key.name.
    string
    hostkey_rsa2048                        RSA certificate used by SSH proxy. Source firewall.ssh.local-key.name.
    string
    untrusted_caname                       Untrusted CA certificate used by SSH Inspection. Source firewall.ssh.local-ca.name.
    string

   string

   boolean                  • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                            • yes ←

   string

   string                 "root"

NOTES                   *fortios_firewall_ssh_setting-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_firewall_ssh_setting-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: SSH proxy settings.
     fortios_firewall_ssh_setting:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       firewall_ssh_setting:
         caname: "<your_own_value> (source firewall.ssh.local-ca.name)"
         host_trusted_checking: "enable"
         hostkey_dsa1024: "myhostname (source firewall.ssh.local-key.name)"
         hostkey_ecdsa256: "myhostname (source firewall.ssh.local-key.name)"
         hostkey_ecdsa384: "myhostname (source firewall.ssh.local-key.name)"
         hostkey_ecdsa521: "myhostname (source firewall.ssh.local-key.name)"
         hostkey_ed25519: "myhostname (source firewall.ssh.local-key.name)"
         hostkey_rsa2048: "myhostname (source firewall.ssh.local-key.name)"
         untrusted_caname: "<your_own_value> (source firewall.ssh.local-ca.name)"
