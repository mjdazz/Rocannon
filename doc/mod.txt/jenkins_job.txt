SYNOPSIS                   *jenkins_job*

     • Manage Jenkins jobs by using Jenkins REST API.

REQUIREMENTS                   *jenkins_job-requirements*

   The below requirements are needed on the host that executes this module.

     • python-jenkins >= 0.4.12

PARAMETERS                   *jenkins_job-parameters*

    Parameter      Choices/Defaults                                                                               Comments
                                        config in XML format.
   config                               Required if job does not yet exist.
   -                                    Mutually exclusive with enabled.
                                        Considered if state=present.
   enabled        Choices:              Whether the job should be enabled or disabled.
   boolean        • no                  Mutually exclusive with config.
                  • yes                 Considered if state=present.
   name                                 Name of the Jenkins job.
   - / required
   password                             Password to authenticate with the Jenkins server.
   -
   state          Choices:
   -              • present ←           Attribute that specifies if the job has to be created or deleted.
                  • absent
   token                                API token used to authenticate alternatively to password.
   -
   url          Default:                URL where the Jenkins server is accessible.
   -            "http://localhost:8080"
   user                                 User to authenticate with the Jenkins server.
   -

EXAMPLES                   *jenkins_job-examples*

 # Create a jenkins job using basic authentication
 - jenkins_job:
     config: "{{ lookup('file', 'templates/test.xml') }}"
     name: test
     password: admin
     url: http://localhost:8080
     user: admin

 # Create a jenkins job using the token
 - jenkins_job:
     config: "{{ lookup('template', 'templates/test.xml.j2') }}"
     name: test
     token: asdfasfasfasdfasdfadfasfasdfasdfc
     url: http://localhost:8080
     user: admin

 # Delete a jenkins job using basic authentication
 - jenkins_job:
     name: test
     password: admin
     state: absent
     url: http://localhost:8080
     user: admin

 # Delete a jenkins job using the token
 - jenkins_job:
     name: test
     token: asdfasfasfasdfasdfadfasfasdfasdfc
     state: absent
     url: http://localhost:8080
     user: admin

 # Disable a jenkins job using basic authentication
 - jenkins_job:
     name: test
     password: admin
     enabled: False
     url: http://localhost:8080
     user: admin

 # Disable a jenkins job using the token
 - jenkins_job:
     name: test
     token: asdfasfasfasdfasdfadfasfasdfasdfc
     enabled: False
     url: http://localhost:8080
     user: admin

RETURN VALUES                   *jenkins_job-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   enabled          Whether the jenkins job is enabled or not.
   boolean success  Sample:
                    True
   name             Name of the jenkins job.
   string  success  Sample:
                    test-job
   state            State of the jenkins job.
   string  success  Sample:
                    present
   url              Url to connect to the Jenkins server.
   string  success  Sample:
                    https://jenkins.mydomain.com
   user             User used for authentication.
   string  success  Sample:
                    admin
