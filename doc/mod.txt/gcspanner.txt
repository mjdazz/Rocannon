DEPRECATED                   *gcspanner*

                                Removed in Ansible:
                  version: 2.12
        Why:      Updated modules released with increased functionality
    Alternative:  Use gcp_spanner_database and/or gcp_spanner_instance instead.

SYNOPSIS                   *gcspanner-synopsis*

     • Create and Delete Instances/Databases on Spanner. See https://cloud.google.com/spanner/docs for an overview.

REQUIREMENTS                   *gcspanner-requirements*

   The below requirements are needed on the host that executes this module.

     • python >= 2.6
     • google-auth >= 0.5.0
     • google-cloud-spanner >= 0.23.0

PARAMETERS                   *gcspanner-parameters*

         Parameter       Choices/Defaults                                                                          Comments
   configuration                          Configuration the instance should use.
   - / required                           Examples are us-central1, asia-east1 and europe-west1.
   database_name                          Name of database contained on the instance.
   -
   force_instance_delete   Choices:
   boolean                 • no ←         To delete an instance, this argument must exist and be true (along with state being equal to absent).
                           • yes
   instance_display_name                  Name of Instance to display.
   -                                      If not specified, instance_id will be used instead.
   instance_id                            GCP spanner instance name.
   - / required
   node_count            Default:         Number of nodes in the instance.
   -                     1
   state                   Choices:       State of the instance or database. Applies to the most granular resource.
   -                       • absent       If a database_name is specified we remove it.
                           • present ←    If only instance_id is specified, that is what is removed.

NOTES                   *gcspanner-notes*

   Note

     • Changing the configuration on an existing instance is not supported.

EXAMPLES                   *gcspanner-examples*

 - name: Create instance
   gcspanner:
     instance_id: '{{ instance_id }}'
     configuration: '{{ configuration }}'
     state: present
     node_count: 1

 - name: Create database
   gcspanner:
     instance_id: '{{ instance_id }}'
     configuration: '{{ configuration }}'
     database_name: '{{ database_name }}'
     state: present

 - name: Delete instance (and all databases)
 - gcspanner:
     instance_id: '{{ instance_id }}'
     configuration: '{{ configuration }}'
     state: absent
     force_instance_delete: yes

RETURN VALUES                   *gcspanner-return values*

   Common return values are documented here, the following are the fields unique to this module:

         Key                                    Returned                                                                                Description
   database_name                                                                       Name of database.
   string          When database name is specified                                     Sample:
                                                                                       mydatabase
   instance_id                                                                         Name of instance.
   string          Always                                                              Sample:
                                                                                       myinstance
   previous_values                                                                     List of dictionaries containing previous values prior to update.
   dictionary      When an instance update has occurred and a field has been modified. Sample:
                                                                                       'previous_values': { 'instance': { 'instance_display_name': 'my-instance', 'node_count': 1 } }
   state                                                                               The state of the instance or database. Value will be either 'absent' or 'present'.
   string          Always                                                              Sample:
                                                                                       present
   updated                                                                             Boolean field to denote an update has occurred.
   boolean         When an update has occurred.                                        Sample:
                                                                                       True
