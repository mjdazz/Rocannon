SYNOPSIS                   *github_release*

     • Fetch metadata about GitHub Releases

REQUIREMENTS                   *github_release-requirements*

   The below requirements are needed on the host that executes this module.

     • github3.py >= 1.0.0a3

PARAMETERS                   *github_release-parameters*

    Parameter     Choices/Defaults                                                                               Comments
   action         Choices:
   - / required   • latest_release   Action to perform
                  • create_release
   body
   -                                 Description of the release when creating a release
   added in 2.4
   draft          Choices:
   boolean        • no ←             Sets if the release is a draft or not. (boolean)
   added in 2.4   • yes
   name
   -                                 Name of release when creating a release
   added in 2.4
   password
   -                                 The GitHub account password for the user. Mutually exclusive with token.
   added in 2.4
   prerelease     Choices:
   boolean        • no ←             Sets if the release is a prerelease or not. (boolean)
   added in 2.4   • yes
   repo                              Repository name
   - / required
   tag
   -                                 Tag name when creating a release. Required when using action is set to create_release.
   added in 2.4
   target
   -                                 Target of release when creating a release
   added in 2.4
   token                             GitHub Personal Access Token for authenticating. Mutually exclusive with password.
   -
   user                              The GitHub account that owns the repository
   - / required

EXAMPLES                   *github_release-examples*

 - name: Get latest release of a public repository
   github_release:
     user: ansible
     repo: ansible
     action: latest_release

 - name: Get latest release of testuseer/testrepo
   github_release:
     token: tokenabc1234567890
     user: testuser
     repo: testrepo
     action: latest_release

 - name: Get latest release of test repo using username and password. Ansible 2.4.
   github_release:
     user: testuser
     password: secret123
     repo: testrepo
     action: latest_release

 - name: Create a new release
   github_release:
     token: tokenabc1234567890
     user: testuser
     repo: testrepo
     action: create_release
     tag: test
     target: master
     name: My Release
     body: Some description

RETURN VALUES                   *github_release-return values*

   Common return values are documented here, the following are the fields unique to this module:

        Key       Returned                                                                                Description
                           Version of the created release
   create_release          For Ansible version 2.5 and later, if specified release version already exists, then State is unchanged
   string         success  For Ansible versions prior to 2.5, if specified release version already exists, then State is skipped
                           Sample:
                           1.1.0
   latest_release          Version of the latest release
   string         success  Sample:
                           1.1.0
