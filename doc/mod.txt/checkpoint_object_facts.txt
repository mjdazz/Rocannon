SYNOPSIS                   *checkpoint_object_facts*

     • Get object facts on Check Point devices. All operations are performed over Web Services API.

PARAMETERS                   *checkpoint_object_facts-parameters*

     Parameter   Choices/Defaults                                                                              Comments
   ip_only         Choices:
   boolean         • no ←         Filter only by IP address.
                   • yes
   object_filter                  Filter expression for search. It accepts AND/OR logical operators and performs a textual and IP address search. To search only by IP address, set ip_only argument
   -                              to True. which can be filtered with the filter argument.
   object_type                    Type of the object to search. Must be a valid API resource name
   string
   uid                            UID of the object. If UID is not provided, it will do a full search which can be filtered with the filter argument.
   -

EXAMPLES                   *checkpoint_object_facts-examples*

 - name: Get object facts
   checkpoint_object_facts:
     object_filter: 192.168.30.30
     ip_only: yes

RETURN VALUES                   *checkpoint_object_facts-return values*

   Common return values are documented here, the following are the fields unique to this module:

        Key      Returned                                                                                 Description
   ansible_hosts always.  The checkpoint object facts.
   list
