SYNOPSIS                   *zabbix_hostmacro*

     • manages Zabbix host macros, it can create, update or delete them.

REQUIREMENTS                   *zabbix_hostmacro-requirements*

   The below requirements are needed on the host that executes this module.

     • python >= 2.6
     • zabbix-api >= 0.5.4

PARAMETERS                   *zabbix_hostmacro-parameters*

        Parameter      Choices/Defaults                                                                           Comments
   force                 Choices:
   boolean               • no           Only updates an existing macro if set to yes.
   added in 2.5          • yes ←
   host_name                            Name of the host.
   string / required
   http_login_password                  Basic Auth password
   string
   http_login_user                      Basic Auth login
   string / required
   login_password                       Zabbix user password.
   string / required
   login_user                           Zabbix user name.
   string / required
   macro_name                           Name of the host macro without the enclosing curly braces and the leading dollar sign.
   string / required
   macro_value                          Value of the host macro.
   string                               Required if state=present.
   server_url                           URL of Zabbix server, with protocol (http or https). url is an alias for server_url.
   string / required                    aliases: url
   state                 Choices:       State of the macro.
   string                • present ←    On present, it will create if macro does not exist or update the macro if the associated data is different.
                         • absent       On absent will remove a macro if it exists.
   timeout             Default:         The timeout of API request (seconds).
   integer             10
   validate_certs        Choices:
   boolean               • no           If set to False, SSL certificates will not be validated. This should only be used on personally controlled sites using self-signed certificates.
   added in 2.5          • yes ←

NOTES                   *zabbix_hostmacro-notes*

   Note

     • If you use login_password=zabbix, the word “zabbix” is replaced by “""""” in all module output, because login_password uses no_log. See this FAQ for more information.

EXAMPLES                   *zabbix_hostmacro-examples*

 - name: Create new host macro or update an existing macro's value
   local_action:
     module: zabbix_hostmacro
     server_url: http://monitor.example.com
     login_user: username
     login_password: password
     host_name: ExampleHost
     macro_name: EXAMPLE.MACRO
     macro_value: Example value
     state: present
