SYNOPSIS                   *pn_vrouter_loopback_interface*

     • This module can be used to add loopback interface to a vRouter or remove loopback interface from a vRouter.

PARAMETERS                   *pn_vrouter_loopback_interface-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   pn_cliswitch                       Target switch to run the CLI on.
   string
   pn_index                           loopback index from 1 to 255.
   string
   pn_ip                              loopback IP address.
   string / required
   pn_vrouter_name                    name of service config.
   string / required
   state               Choices:
   string              • present ←    State the action to perform. Use present to add vrouter-loopback-interface and absent to remove vrouter-loopback-interface.
                       • absent

EXAMPLES                   *pn_vrouter_loopback_interface-examples*

 - name: Add vrouter loopback interface
   pn_vrouter_loopback_interface:
     state: "present"
     pn_cliswitch: "sw01"
     pn_vrouter_name: "sw01-vrouter"
     pn_ip: "192.168.10.1"

 - name: Remove vrouter loopback interface
   pn_vrouter_loopback_interface:
     state: "absent"
     pn_cliswitch: "sw01"
     pn_vrouter_name: "sw01-vrouter"
     pn_ip: "192.168.10.1"
     pn_index: "2"

RETURN VALUES                   *pn_vrouter_loopback_interface-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   indicates whether the CLI caused changes on the target.
   boolean
   command always   the CLI command run on the target node.
   string
   stderr  on error set of error response from the vrouter-loopback-interface command.
   list
   stdout  always   set of responses from the vrouter-loopback-interface command.
   list
