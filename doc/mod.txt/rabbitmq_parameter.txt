SYNOPSIS                   *rabbitmq_parameter*

     • Manage dynamic, cluster-wide parameters for RabbitMQ

PARAMETERS                   *rabbitmq_parameter-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   component                     Name of the component of which the parameter is being set
   - / required
   name                          Name of the parameter being set
   - / required
   node         Default:         erlang node name of the rabbit we wish to configure
   -            "rabbit"
   state          Choices:
   -              • present ←    Specify if user is to be added or removed
                  • absent
   value                         Value of the parameter, as a JSON term
   -
   vhost        Default:         vhost to apply access privileges.
   -            "/"

EXAMPLES                   *rabbitmq_parameter-examples*

 # Set the federation parameter 'local_username' to a value of 'guest' (in quotes)
 - rabbitmq_parameter:
     component: federation
     name: local-username
     value: '"guest"'
     state: present
