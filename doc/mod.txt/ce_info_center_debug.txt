SYNOPSIS                   *ce_info_center_debug*

     • Manages information center debug configurations on HUAWEI CloudEngine switches.

PARAMETERS                   *ce_info_center_debug-parameters*

      Parameter          Choices/Defaults                                                                             Comments
   channel_id                                  Number of a channel. The value is an integer ranging from 0 to 9. The default value is 0.
   -
                      Choices:
   debug_enable       • no_use ←               Whether a device is enabled to output debugging information.
   -                  • true
                      • false
                      Choices:
                      • emergencies
                      • alert
   debug_level        • critical
   -                  • error                  Debug level permitted to output.
                      • warning
                      • notification
                      • informational
                      • debugging
                      Choices:
                      • date_boot
                      • date_second
                      • date_tenthsecond
   debug_time_stamp   • date_millisecond
   -                  • shortdate_second       Timestamp type of debugging information.
                      • shortdate_tenthsecond
                      • shortdate_millisecond
                      • formatdate_second
                      • formatdate_tenthsecond
                      • formatdate_millisecond
   module_name                                 Module name of the rule. The value is a string of 1 to 31 case-insensitive characters. The default value is default. Please use lower-case letter,
   -                                           such as [aaa, acl, arp, bfd].
   state              Choices:
   -                  • present ←              Specify desired state of the resource.
                      • absent

NOTES                   *ce_info_center_debug-notes*

   Note

     • This module requires the netconf system service be enabled on the remote device being managed.
     • Recommended connection is netconf.
     • This module also works with local connections for legacy playbooks.

EXAMPLES                   *ce_info_center_debug-examples*

 - name: CloudEngine info center debug test
   hosts: cloudengine
   connection: local
   gather_facts: no
   vars:
     cli:
       host: "{{ inventory_hostname }}"
       port: "{{ ansible_ssh_port }}"
       username: "{{ username }}"
       password: "{{ password }}"
       transport: cli

   tasks:

   - name: "Config debug time stamp"
     ce_info_center_debug:
       state: present
       debug_time_stamp: date_boot
       provider: "{{ cli }}"

   - name: "Undo debug time stamp"
     ce_info_center_debug:
       state: absent
       debug_time_stamp: date_boot
       provider: "{{ cli }}"

   - name: "Config debug module log level"
     ce_info_center_debug:
       state: present
       module_name: aaa
       channel_id: 1
       debug_enable: true
       debug_level: error
       provider: "{{ cli }}"

   - name: "Undo debug module log level"
     ce_info_center_debug:
       state: absent
       module_name: aaa
       channel_id: 1
       debug_enable: true
       debug_level: error
       provider: "{{ cli }}"

RETURN VALUES                   *ce_info_center_debug-return values*

   Common return values are documented here, the following are the fields unique to this module:

      Key     Returned                                                                                  Description
   changed             check to see if a change was made on the device
   boolean    always   Sample:
                       True
   end_state           k/v pairs of aaa params after module execution
   dictionary always   Sample:
                       {'debugTimeStamp': 'DATE_BOOT'}
   existing            k/v pairs of existing aaa server
   dictionary always   Sample:
                       {'debugTimeStamp': 'DATE_MILLISECOND'}
   proposed            k/v pairs of parameters passed into module
   dictionary always   Sample:
                       {'state': 'present', 'debug_time_stamp': 'date_boot'}
   updates             command sent to the device
   list       always   Sample:
                       ['info-center timestamp debugging boot']
