DEPRECATED                   *purefa_facts*

                   Removed in Ansible:
                  version: 2.13
        Why:      Deprecated in favor of _info module.
    Alternative:  Use purefa_info instead.

SYNOPSIS                   *purefa_facts-synopsis*

     • Collect facts information from a Pure Storage Flasharray running the Purity//FA operating system. By default, the module will collect basic fact information including hosts, host groups,
       protection groups and volume counts. Additional fact information can be collected based on the configured set of arguments.

REQUIREMENTS                   *purefa_facts-requirements*

   The below requirements are needed on the host that executes this module.

     • python >= 2.7
     • purestorage

PARAMETERS                   *purefa_facts-parameters*

     Parameter   Choices/Defaults                                                                              Comments
   api_token
   string /                       FlashArray API token for admin privileged user.
   required
   fa_url
   string /                       FlashArray management IPv4 address or Hostname.
   required
   gather_subset Default:         When supplied, this argument will define the facts to be collected. Possible values for this include all, minimum, config, performance, capacity, network, subnet,
   list          "minimum"        interfaces, hgroups, pgroups, hosts, admins, volumes, snapshots, pods, vgroups, offload, apps and arrays.

NOTES                   *purefa_facts-notes*

   Note

     • This module requires the purestorage Python library
     • You must set PUREFA_URL and PUREFA_API environment variables if fa_url and api_token arguments are not passed to the module directly

EXAMPLES                   *purefa_facts-examples*

 - name: collect default set of facts
   purefa_facts:
     fa_url: 10.10.10.2
     api_token: e31060a7-21fc-e277-6240-25983c6c4592

 - name: collect configuration and capacity facts
   purefa_facts:
     gather_subset:
       - config
       - capacity
     fa_url: 10.10.10.2
     api_token: e31060a7-21fc-e277-6240-25983c6c4592

 - name: collect all facts
   purefa_facts:
     gather_subset:
       - all
     fa_url: 10.10.10.2
     api_token: e31060a7-21fc-e277-6240-25983c6c4592
