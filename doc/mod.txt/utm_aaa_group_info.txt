SYNOPSIS                   *utm_aaa_group_info*

     • get info for a reverse_proxy frontend entry in SOPHOS UTM.

PARAMETERS                   *utm_aaa_group_info-parameters*

     Parameter    Choices/Defaults                                                                              Comments
   headers                         A dictionary of additional headers to be sent to POST and PUT requests.
   dictionary                      Is needed for some modules
   name                            The name of the object. Will be used to identify the entry
   - / required
   state            Choices:       The desired state of the object.
   string           • absent       present will create or update an object
                    • present ←    absent will delete an object if it was present
   utm_host
   string /                        The REST Endpoint of the Sophos UTM.
   required
   utm_port       Default:         The port of the REST interface.
   integer        4444
   utm_protocol     Choices:
   string           • http         The protocol of the REST Endpoint.
                    • https ←
   utm_token                       The token used to identify at the REST-API. See https://www.sophos.com/en-us/medialibrary/PDFs/documentation/UTMonAWS/Sophos-UTM-RESTful-API.pdf?la=en, Chapter
   string /                        2.4.2.
   required
   validate_certs   Choices:
   boolean          • no           Whether the REST interface's ssl certificate should be verified or not.
                    • yes ←

EXAMPLES                   *utm_aaa_group_info-examples*

 - name: Remove UTM aaa_group

     utm_host: sophos.host.name
