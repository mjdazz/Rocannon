SYNOPSIS                   *xbps*

     • Manage packages with the XBPS package manager.

PARAMETERS                   *xbps-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   name                          Name of the package to install, upgrade, or remove.
   -
   recurse        Choices:
   boolean        • no ←         When removing a package, also remove its dependencies, provided that they are not required by other packages and were not explicitly installed by a user.
                  • yes
                  Choices:
   state          • present ←    Desired state of the package.
   -              • absent
                  • latest
   update_cache   Choices:
   boolean        • no           Whether or not to refresh the master package lists. This can be run as part of a package installation or as a separate step.
                  • yes ←
   upgrade        Choices:
   boolean        • no ←         Whether or not to upgrade whole system
                  • yes

EXAMPLES                   *xbps-examples*

 # Install package foo
 - xbps: name=foo state=present
 # Upgrade package foo
 - xbps: name=foo state=latest update_cache=yes
 # Remove packages foo and bar
 - xbps: name=foo,bar state=absent
 # Recursively remove package foo
 - xbps: name=foo state=absent recurse=yes
 # Update package cache
 - xbps: update_cache=yes
 # Upgrade packages
 - xbps: upgrade=yes

RETURN VALUES                   *xbps-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key    Returned                                                                                   Description
   msg               Message about results
   string   success  Sample:
                     System Upgraded
   packages          Packages that are affected/would be affected
   list     success  Sample:
                     ['ansible']
