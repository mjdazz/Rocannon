DEPRECATED                   *nginx_status_facts*

                    Removed in Ansible:
                  version: 2.13
        Why:      Deprecated in favour of _info module.
    Alternative:  Use nginx_status_info instead.

SYNOPSIS                   *nginx_status_facts-synopsis*

     • Gathers facts from nginx from an URL having stub_status enabled.

PARAMETERS                   *nginx_status_facts-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   timeout      Default:         HTTP connection timeout in seconds.
   -            10
   url                           URL of the nginx status.
   - / required

NOTES                   *nginx_status_facts-notes*

   Note

     • See http://nginx.org/en/docs/http/ngx_http_stub_status_module.html for more information.

EXAMPLES                   *nginx_status_facts-examples*

 # Gather status facts from nginx on localhost
 - name: get current http stats
   nginx_status_facts:
     url: http://localhost/nginx_status

 # Gather status facts from nginx on localhost with a custom timeout of 20 seconds
 - name: get current http stats
   nginx_status_facts:
     url: http://localhost/nginx_status
     timeout: 20

RETURN VALUES                   *nginx_status_facts-return values*

   Common return values are documented here, the following are the fields unique to this module:

                    Key                  Returned                                                                     Description
   nginx_status_facts.accepts                     The total number of accepted client connections.
   integer                               success  Sample:
                                                  81769947
   nginx_status_facts.active_connections          Active connections.
   integer                               success  Sample:
                                                  2340
   nginx_status_facts.data                        HTTP response as is.
   string                                success  Sample:
                                                  Active connections: 2340 server accepts handled requests 81769947 81769947 144332345 Reading: 0 Writing: 241 Waiting: 2092
   nginx_status_facts.handled                     The total number of handled connections. Generally, the parameter value is the same as accepts unless some resource limits have been reached.
   integer                               success  Sample:
                                                  81769947
   nginx_status_facts.reading            success  The current number of connections where nginx is reading the request header.
   integer
   nginx_status_facts.requests                    The total number of client requests.
   integer                               success  Sample:
                                                  144332345
   nginx_status_facts.waiting                     The current number of idle client connections waiting for a request.
   integer                               success  Sample:
                                                  2092
   nginx_status_facts.writing                     The current number of connections where nginx is writing the response back to the client.
   integer                               success  Sample:
                                                  241
