SYNOPSIS                   *pacemaker_cluster*

     • This module can manage a pacemaker cluster and nodes from Ansible using the pacemaker cli.

PARAMETERS                   *pacemaker_cluster-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   force          Choices:
   boolean        • no           Force the change of the cluster state
                  • yes ←
   node                          Specify which node of the cluster you want to manage. None == the cluster status itself, 'all' == check the status of all nodes.
   -
                  Choices:
   state          • cleanup
   - / required   • offline      Indicate desired state of the cluster
                  • online
                  • restart
   timeout      Default:         Timeout when the module should considered that the action has failed
   -            300

EXAMPLES                   *pacemaker_cluster-examples*

 ---
 - name: Set cluster Online
   hosts: localhost
   gather_facts: no
   tasks:
   - name: Get cluster state
     pacemaker_cluster:
       state: online

RETURN VALUES                   *pacemaker_cluster-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   True if the cluster state has changed
   boolean
   out              The output of the current state of the cluster. It return a list of the nodes state.
   string  always   Sample:
                    out: [[" overcloud-controller-0", " Online"]]}
   rc      always   exit code of the module
   boolean
