SYNOPSIS                   *ovh_ip_loadbalancing_backend*

     • Manage OVH (French European hosting provider) LoadBalancing IP backends

REQUIREMENTS                   *ovh_ip_loadbalancing_backend-requirements*

   The below requirements are needed on the host that executes this module.

     • ovh > 0.3.5

PARAMETERS                   *ovh_ip_loadbalancing_backend-parameters*

       Parameter      Choices/Defaults                                                                            Comments
   application_key                     The applicationKey to use
   - / required
   application_secret                  The application secret to use
   - / required
   backend                             The IP address of the backend to update / modify / delete
   - / required
   consumer_key                        The consumer key to use
   - / required
   endpoint                            The endpoint to use ( for instance ovh-eu)
   - / required
   name                                Name of the LoadBalancing internal name (ip-X.X.X.X)
   - / required
                        Choices:
   probe                • none ←
   -                    • http         Determines the type of probe to use for this backend
                        • icmp
                        • oco
   state                Choices:
   -                    • present ←    Determines whether the backend is to be created/modified or deleted
                        • absent
   timeout            Default:         The timeout in seconds used to wait for a task to be completed.
   -                  120
   weight             Default:         Determines the weight for this backend
   -                  8

NOTES                   *ovh_ip_loadbalancing_backend-notes*

   Note

     • Uses the python OVH Api https://github.com/ovh/python-ovh. You have to create an application (a key and secret) with a consumer key as described into
       https://eu.api.ovh.com/g934.first_step_with_api

EXAMPLES                   *ovh_ip_loadbalancing_backend-examples*

 # Adds or modify the backend '212.1.1.1' to a
 # loadbalancing 'ip-1.1.1.1'
 - ovh_ip_loadbalancing:
     name: ip-1.1.1.1
     backend: 212.1.1.1
     state: present
     probe: none
     weight: 8
     endpoint: ovh-eu
     application_key: yourkey
     application_secret: yoursecret
     consumer_key: yourconsumerkey

 # Removes a backend '212.1.1.1' from a loadbalancing 'ip-1.1.1.1'
 - ovh_ip_loadbalancing:
     name: ip-1.1.1.1
     backend: 212.1.1.1
     state: absent
     endpoint: ovh-eu
     application_key: yourkey
     application_secret: yoursecret
     consumer_key: yourconsumerkey
