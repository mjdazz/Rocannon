SYNOPSIS                   *win_psmodule*

     • This module helps to install Windows PowerShell modules and register custom modules repository on Windows-based systems.

PARAMETERS                   *win_psmodule-parameters*

        Parameter       Choices/Defaults                                                                           Comments
   allow_clobber          Choices:
   boolean                • no ←         If yes allows install modules that contains commands those have the same names as commands that already exists.
                          • yes
   allow_prerelease       Choices:       If yes installs modules marked as prereleases.
   boolean                • no ←         It doesn't work with the parameters minimum_version and/or maximum_version.
   added in 2.8           • yes          It doesn't work with the state set to absent.
   maximum_version
   string                                The maximum version of the PowerShell module that has to be installed.
   added in 2.8
   minimum_version
   string                                The minimum version of the PowerShell module that has to be installed.
   added in 2.8
   name                                  Name of the Windows PowerShell module that has to be installed.
   string / required
   repository                            Name of the custom repository to use.
   string
   required_version
   string                                The exact version of the PowerShell module that has to be installed.
   added in 2.8
   skip_publisher_check   Choices:       If yes, allows you to install a different version of a module that already exists on your computer in the case when a different one is not digitally signed
   boolean                • no ←         by a trusted publisher and the newest existing module is digitally signed by a trusted publisher.
   added in 2.8           • yes
                          Choices:       If present a new module is installed.
   state                  • absent       If absent a module is removed.
   string                 • latest       If latest a module is updated to the newest version. This option was added in version 2.8.

   url                                   URL of the custom repository to register.

NOTES                   *win_psmodule-notes*

   Note

     • PowerShell modules needed - PowerShellGet >= 1.6.0 - PackageManagement >= 1.1.7
     • PowerShell package provider needed - NuGet >= 2.8.5.201
     • On PowerShell 5.x required modules and a package provider will be updated under the first run of the win_psmodule module.
     • On PowerShell 3.x and 4.x you have to install them before using the win_psmodule.

EXAMPLES                   *win_psmodule-examples*

 ---
 - name: Add a PowerShell module
   win_psmodule:
     name: PowerShellModule
     state: present

 - name: Add an exact version of PowerShell module
   win_psmodule:
     name: PowerShellModule
     required_version: "4.0.2"
     state: present

 - name: Install or update an existing PowerShell module to the newest version
   win_psmodule:
     name: PowerShellModule
     state: latest

 - name: Install newer version of built-in Windows module
   win_psmodule:
     name: Pester
     skip_publisher_check: yes
     state: present

 - name: Add a PowerShell module and register a repository
   win_psmodule:
     name: MyCustomModule
     repository: MyRepository
     state: present

 - name: Add a PowerShell module from a specific repository
   win_psmodule:
     name: PowerShellModule
     repository: MyRepository
     state: present

 - name: Remove a PowerShell module
   win_psmodule:
     name: PowerShellModule
     state: absent
