SYNOPSIS                   *win_iis_website*

     • Creates, Removes and configures a IIS Web site.

PARAMETERS                   *win_iis_website-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   application_pool                   The application pool in which the new site executes.
   string
   hostname                           The host header to bind to / use for the new site.
   string
   ip                                 The IP address to bind to / use for the new site.
   string
   name                               Names of web site.
   string / required
   parameters                         Custom site Parameters from string where properties are separated by a pipe and property name/values by colon Ex. "foo:1|bar:2"
   string
   physical_path                      The physical path on the remote host to use for the new site.
   string                             The specified folder must already exist.
   port                               The port to bind to / use for the new site.
   integer
   site_id                            Explicitly set the IIS numeric ID for a site.
   string                             Note that this value cannot be changed after the website has been created.
   ssl                                Enables HTTPS binding on the site..
   string
                       Choices:
   state               • absent
   string              • started      State of the web site
                       • stopped
                       • restarted

EXAMPLES                   *win_iis_website-examples*

 # Start a website

 - name: Acme IIS site
   win_iis_website:
     name: Acme
     state: started
     port: 80
     ip: 127.0.0.1
     hostname: acme.local
     application_pool: acme
     physical_path: C:\sites\acme
     parameters: logfile.directory:C:\sites\logs
   register: website

 # Remove Default Web Site and the standard port 80 binding
 - name: Remove Default Web Site
   win_iis_website:
     name: "Default Web Site"
     state: absent

 # Some commandline examples:

 # This return information about an existing host
 # $ ansible -i vagrant-inventory -m win_iis_website -a "name='Default Web Site'" window
 # host | success >> {
 #     "changed": false,
 #     "site": {
 #         "ApplicationPool": "DefaultAppPool",
 #         "Bindings": [
 #             "":80:"
 #         ],
 #         "ID": 1,
 #         "Name": "Default Web Site",
 #         "PhysicalPath": "%SystemDrive%\\inetpub\\wwwroot",
 #         "State": "Stopped"
 #     }
 # }

 # This stops an existing site.
 # $ ansible -i hosts -m win_iis_website -a "name='Default Web Site' state=stopped" host

 # This creates a new site.
 # $ ansible -i hosts -m win_iis_website -a "name=acme physical_path=C:\\sites\\acme" host

 # Change logfile.
 # $ ansible -i hosts -m win_iis_website -a "name=acme physical_path=C:\\sites\\acme" host
