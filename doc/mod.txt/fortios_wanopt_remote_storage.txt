SYNOPSIS                   *fortios_wanopt_remote_storage*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify wanopt feature and remote_storage category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_wanopt_remote_storage-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_wanopt_remote_storage-parameters*

         Parameter       Choices/Defaults                                                                          Comments
   host                                   FortiOS or FortiGate IP address.
   string
   https                   Choices:
   boolean                 • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                           • yes ←
   password              Default:         FortiOS or FortiGate password.
   string                ""
   ssl_verify              Choices:
   boolean                 • no           Ensures FortiGate certificate must be verified by a proper CA.
                           • yes ←
   username                               FortiOS or FortiGate username.
   string
   vdom                  Default:         Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different unit.
   string                "root"
   wanopt_remote_storage Default:         Configure a remote cache device as Web cache storage.
   dictionary            null
       local_cache_id                     ID that this device uses to connect to the remote device.
       string
       remote_cache_id                    ID of the remote device to which the device connects.
       string
       remote_cache_ip                    IP address of the remote device to which the device connects.
       string
       status              Choices:
       string              • disable      Enable/disable using remote device as Web cache storage.
                           • enable

NOTES                   *fortios_wanopt_remote_storage-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_wanopt_remote_storage-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure a remote cache device as Web cache storage.
     fortios_wanopt_remote_storage:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       wanopt_remote_storage:
         local_cache_id: "<your_own_value>"
         remote_cache_id: "<your_own_value>"
         remote_cache_ip: "<your_own_value>"
         status: "disable"
