SYNOPSIS                   *fortios_system_network_visibility*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify system feature and network_visibility category. Examples include all parameters
       and values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_system_network_visibility-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_system_network_visibility-parameters*

              Parameter             Choices/Defaults                                                                     Comments
   host                                              FortiOS or FortiGate IP address.
   string
   https                              Choices:
   boolean                            • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                                      • yes ←
   password                         Default:         FortiOS or FortiGate password.
   string                           ""
   ssl_verify                         Choices:
   boolean                            • no           Ensures FortiGate certificate must be verified by a proper CA.
                                      • yes ←
   system_network_visibility        Default:         Configure network visibility settings.
   dictionary                       null
    destination_hostname_visibility   Choices:
    string                            • disable      Enable/disable logging of destination hostname visibility.
                                      • enable
    destination_location              Choices:
    string                            • disable      Enable/disable logging of destination geographical location visibility.
                                      • enable
    destination_visibility            Choices:
    string                            • disable      Enable/disable logging of destination visibility.
                                      • enable
    hostname_limit                                   Limit of the number of hostname table entries (0 - 50000).
    integer
    hostname_ttl                                     TTL of hostname table entries (60 - 86400).
    integer
    source_location                   Choices:
    string                            • disable      Enable/disable logging of source geographical location visibility.
                                      • enable
   username                                          FortiOS or FortiGate username.
   string
   vdom                             Default:         Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different
   string                           "root"           unit.

NOTES                   *fortios_system_network_visibility-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_system_network_visibility-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure network visibility settings.
     fortios_system_network_visibility:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       system_network_visibility:
         destination_hostname_visibility: "disable"
         destination_location: "disable"
         destination_visibility: "disable"
         hostname_limit: "6"
         hostname_ttl: "7"
         source_location: "disable"
