SYNOPSIS                   *selogin*

     • Manages linux user to SELinux user mapping

REQUIREMENTS                   *selogin-requirements*

   The below requirements are needed on the host that executes this module.

     • libselinux
     • policycoreutils

PARAMETERS                   *selogin-parameters*

        Parameter       Choices/Defaults                                                                           Comments
   ignore_selinux_state   Choices:
   boolean                • no ←         Run independent of selinux runtime state
                          • yes
   login                                 a Linux user
   - / required
   reload               Default:         Reload SELinux policy after commit.
   -                    "yes"
   selevel              Default:         MLS/MCS Security Range (MLS/MCS Systems only) SELinux Range for SELinux login mapping defaults to the SELinux user record range.
   -                    "s0"             aliases: serange
   seuser                                SELinux user name
   - / required
   state                  Choices:
   - / required           • present ←    Desired mapping value.
                          • absent

NOTES                   *selogin-notes*

   Note

     • The changes are persistent across reboots
     • Not tested on any debian based system

EXAMPLES                   *selogin-examples*

 # Modify the default user on the system to the guest_u user
 - selogin:
     login: __default__
     seuser: guest_u
     state: present

 # Assign gijoe user on an MLS machine a range and to the staff_u user
 - selogin:
     login: gijoe
     seuser: staff_u
     serange: SystemLow-Secret
     state: present

 # Assign all users in the engineering group to the staff_u user
 - selogin:
     login: '%engineering'
     seuser: staff_u
     state: present
