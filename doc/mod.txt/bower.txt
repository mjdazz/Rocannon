SYNOPSIS                   *bower*

     • Manage bower packages with bower

PARAMETERS                   *bower-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   name                               The name of a bower package to install
   -
   offline             Choices:
   boolean             • no ←         Install packages from local cache, if the packages were installed before
                       • yes
   path                               The base path where to install the bower packages
   - / required
   production          Choices:
   boolean             • no ←         Install with --production flag
                       • yes
   relative_execpath                  Relative path to bower executable from install path
   -
                       Choices:
   state               • present ←    The state of the bower package
   -                   • absent
                       • latest
   version                            The version to be installed
   -

EXAMPLES                   *bower-examples*

 - name: Install "bootstrap" bower package.
   bower:
     name: bootstrap

 - name: Install "bootstrap" bower package on version 3.1.1.
   bower:
     name: bootstrap
     version: '3.1.1'

 - name: Remove the "bootstrap" bower package.
   bower:
     name: bootstrap
     state: absent

 - name: Install packages based on bower.json.
   bower:
     path: /app/location

 - name: Update packages based on bower.json to their latest version.
   bower:
     path: /app/location
     state: latest

 # install bower locally and run from there
 - npm:
     path: /app/location
     name: bower
     global: no
 - bower:
     path: /app/location
     relative_execpath: node_modules/.bin
