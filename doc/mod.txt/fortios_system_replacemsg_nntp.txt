SYNOPSIS                   *fortios_system_replacemsg_nntp*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify system_replacemsg feature and nntp category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_system_replacemsg_nntp-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

NOTES                   *fortios_system_replacemsg_nntp-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_system_replacemsg_nntp-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Replacement messages.
     fortios_system_replacemsg_nntp:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       state: "present"
       system_replacemsg_nntp:
         buffer: "<your_own_value>"
         format: "none"
         header: "none"
         msg_type: "<your_own_value>"
