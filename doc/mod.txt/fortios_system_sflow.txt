SYNOPSIS                   *fortios_system_sflow*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify system feature and sflow category. Examples include all parameters and values need
       to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_system_sflow-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_system_sflow-parameters*

      Parameter    Choices/Defaults                                                                             Comments
   host                             FortiOS or FortiGate IP address.
   string
   https             Choices:
   boolean           • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                     • yes ←
   password        Default:         FortiOS or FortiGate password.
   string          ""
   ssl_verify        Choices:
   boolean           • no           Ensures FortiGate certificate must be verified by a proper CA.
                     • yes ←
   system_sflow    Default:         Configure sFlow.
   dictionary      null
    collector_ip                    IP address of the sFlow collector that sFlow agents added to interfaces in this VDOM send sFlow datagrams to .
    string
    collector_port                  UDP port number used for sending sFlow datagrams (configure only if required by your sFlow collector or your network configuration) (0 - 65535).
    integer
    source_ip                       Source IP address for sFlow agent.
    string
   username                         FortiOS or FortiGate username.
   string
   vdom            Default:         Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different unit.
   string          "root"

NOTES                   *fortios_system_sflow-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_system_sflow-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure sFlow.
     fortios_system_sflow:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       system_sflow:
         collector_ip: "<your_own_value>"
         collector_port: "4"
         source_ip: "84.230.14.43"
