SYNOPSIS                   *pn_vtep*

     • This module can be used to create a vtep and delete a vtep.

PARAMETERS                   *pn_vtep-parameters*

        Parameter       Choices/Defaults                                                                           Comments
   pn_cliswitch                          Target switch to run the CLI on.
   string
   pn_ip                                 Primary IP address.
   string
   pn_location                           switch name.
   string
   pn_name                               vtep name.
   string
   pn_switch_in_cluster   Choices:
   boolean                • no           Tells whether switch in cluster or not.
                          • yes ←
   pn_virtual_ip                         Virtual/Secondary IP address.
   string
   pn_vrouter_name                       name of the vrouter service.
   string
   state                  Choices:
   string                 • present ←    vtep configuration command.
                          • absent

EXAMPLES                   *pn_vtep-examples*

 - name: create vtep
   pn_vtep:
     pn_cliswitch: 'sw01'
     pn_name: 'foo'
     pn_vrouter_name: 'foo-vrouter'
     pn_ip: '22.22.22.2'
     pn_location: 'sw01'
     pn_virtual_ip: "22.22.22.1"

 - name: delete vtep
   pn_vtep:
     pn_cliswitch: 'sw01'
     state: 'absent'
     pn_name: 'foo'

RETURN VALUES                   *pn_vtep-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   indicates whether the CLI caused changes on the target.
   boolean
   command always   the CLI command run on the target node.
   string
   stderr  on error set of error responses from the vtep command.
   list
   stdout  always   set of responses from the vtep command.
   list
