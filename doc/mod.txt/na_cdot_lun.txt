DEPRECATED                   *na_cdot_lun*

                            Removed in Ansible:
                  version: 2.11
        Why:      Updated modules released with increased functionality
    Alternative:  Use na_ontap_lun instead.

SYNOPSIS                   *na_cdot_lun-synopsis*

     • Create, destroy, resize luns on NetApp cDOT.

REQUIREMENTS                   *na_cdot_lun-requirements*

   The below requirements are needed on the host that executes this module.

     • A physical or virtual clustered Data ONTAP system. The modules were developed with Clustered Data ONTAP 8.3
     • Ansible 2.2
     • netapp-lib (2015.9.25). Install using ‘pip install netapp-lib’

PARAMETERS                   *na_cdot_lun-parameters*

        Parameter      Choices/Defaults                                                                           Comments
   flexvol_name                         The name of the FlexVol the lun should exist on.
   -                                    Required when state=present.
   force_remove        Default:         If "true", override checks that prevent a LUN from being destroyed if it is online and mapped.
   -                   "no"             If "false", destroying an online and mapped LUN will fail.
   force_remove_fenced Default:         If "true", override checks that prevent a LUN from being destroyed while it is fenced.
   -                   "no"             If "false", attempting to destroy a fenced LUN will fail.
                                        The default if not specified is "false". This field is available in Data ONTAP 8.2 and later.

NOTES                   *na_cdot_lun-notes*

   Note

     • The modules prefixed with na\_cdot are built to support the ONTAP storage platform.

EXAMPLES                   *na_cdot_lun-examples*

 - name: Create LUN
   na_cdot_lun:
     state: present
     name: ansibleLUN
     flexvol_name: ansibleVolume
     vserver: ansibleVServer
     size: 5
     size_unit: mb
     hostname: "{{ netapp_hostname }}"
     username: "{{ netapp_username }}"
     password: "{{ netapp_password }}"

 - name: Resize Lun
   na_cdot_lun:
     state: present
     name: ansibleLUN
     force_resize: True
     flexvol_name: ansibleVolume
     vserver: ansibleVServer
     size: 5
     size_unit: gb
     hostname: "{{ netapp_hostname }}"
     username: "{{ netapp_username }}"
     password: "{{ netapp_password }}"
