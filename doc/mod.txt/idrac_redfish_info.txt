SYNOPSIS                   *idrac_redfish_info*

     • Builds Redfish URIs locally and sends them to remote iDRAC controllers to get information back.
     • For use with Dell iDRAC operations that require Redfish OEM extensions
     • This module was called idrac_redfish_facts before Ansible 2.9, returning ansible_facts. Note that the idrac_redfish_info module no longer returns ansible_facts!

   Aliases: idrac_redfish_facts

PARAMETERS                   *idrac_redfish_info-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   baseuri                            Base URI of iDRAC controller
   string / required
   category                           Category to execute on iDRAC controller
   string / required
   command                            List of commands to execute on iDRAC controller
   list / required
   password                           Password for authentication with iDRAC controller
   string / required
   timeout           Default:         Timeout in seconds for URL requests to OOB controller
   integer           10
   username                           User for authentication with iDRAC controller
   string / required

EXAMPLES                   *idrac_redfish_info-examples*

 - name: Get Manager attributes with a default of 20 seconds
   idrac_redfish_command:
     category: Manager
     command: GetManagerAttributes
     baseuri: "{{ baseuri }}"
     username: "{{ username }}"
     password: "{{ password }}"
     timeout: 20

RETURN VALUES                   *idrac_redfish_info-return values*

   Common return values are documented here, the following are the fields unique to this module:

      Key     Returned                                                                                  Description
   msg                 different results depending on task
   dictionary always   Sample:
                       List of Manager attributes
