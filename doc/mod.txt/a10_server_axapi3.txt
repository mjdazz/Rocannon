SYNOPSIS                   *a10_server_axapi3*

     • Manage SLB (Server Load Balancer) server objects on A10 Networks devices via aXAPIv3.

Parameters¶

      Parameter     Choices/Defaults                                                                             Comments
   client_cert                        PEM formatted certificate chain file to be used for SSL client authentication.
   path                               This file can also include the key as well, and if the key is included, client_key is not required.
   client_key                         PEM formatted file that contains your private key to be used for SSL client authentication.
   path                               If client_cert contains both the certificate and key, this option is not required.
   force              Choices:        If yes do not get a cached copy.
   boolean            • no ←          Alias thirsty has been deprecated and will be removed in 2.13.
                      • yes           aliases: thirsty
   force_basic_auth   Choices:
   boolean            • no ←          Credentials specified with url_username and url_password should be passed in HTTP Header.
                      • yes
   host
   string /                           Hostname or IP of the A10 Networks device.
   required
   http_agent       Default:          Header to identify as, generally appears in web server logs.
   string           "ansible-httpget"
                      Choices:
   operation          • create ←      Create, Update or Remove SLB server. For create and update operation, we use the IP address and server name specified in the POST message. For delete
   -                  • update        operation, we use the server name in the request URI.
                      • remove
   password                           Password for the username account.
   string /                           aliases: pass, pwd
   required
   server_ip                          The SLB (Server Load Balancer) server IPv4 address.
   - / required                       aliases: ip, address
   server_name                        The SLB (Server Load Balancer) server name.
   - / required                       aliases: server
   server_ports                       A list of ports to create for the server. Each list item should be a dictionary which specifies the port: and protocol:.
   -                                  aliases: port
   server_status      Choices:        The SLB (Server Load Balancer) virtual server status.
   -                  • enable ←      aliases: action
                      • disable
   url                                HTTP, HTTPS, or FTP URL in the form (http|https|ftp)://[user[:pass]]@host.domain[:port]/path
   string
   url_password                       The password for use in HTTP basic authentication.
   string                             If the url_username parameter is not specified, the url_password parameter will not be used.
   url_username                       The username for use in HTTP basic authentication.
   string                             This parameter can be used without url_password for sites that allow empty passwords
   use_proxy          Choices:
   boolean            • no            If no, it will not use a proxy, even if one is defined in an environment variable on the target hosts.
                      • yes ←
   username                           An account with administrator privileges.
   string /                           aliases: admin, user
   required
   validate_certs     Choices:
   boolean            • no            If no, SSL certificates will not be validated. This should only be used on personally controlled devices using self-signed certificates.
                      • yes ←
   write_config       Choices:        If yes, any changes will cause a write of the running configuration to non-volatile memory. This will save all configuration changes, including those that may
   boolean            • no ←          have been made manually or through other modules, so care should be taken when specifying yes.
                      • yes

Notes¶

   Note

     • Requires A10 Networks aXAPI 2.1.

Examples¶

 # Create a new server
 - a10_server:
     host: a10.mydomain.com
     username: myadmin
     password: mypassword
     server: test
     server_ip: 1.1.1.100
     validate_certs: false
     server_status: enable
     write_config: yes
     operation: create
     server_ports:
       - port-number: 8080
         protocol: tcp
         action: enable
       - port-number: 8443
         protocol: TCP
