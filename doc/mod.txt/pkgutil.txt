SYNOPSIS                   *pkgutil*

     • Manages CSW packages (SVR4 format) on Solaris 10 and 11.
     • These were the native packages on Solaris <= 10 and are available as a legacy feature in Solaris 11.
     • Pkgutil is an advanced packaging system, which resolves dependency on installation. It is designed for CSW packages.

PARAMETERS                   *pkgutil-parameters*

     Parameter    Choices/Defaults                                                                              Comments
   name                            Package name, e.g. (CSWnrpe)
   - / required
   site                            Specifies the repository path to install the package from.
   -                               Its global definition is done in /etc/opt/csw/pkgutil.conf.
                    Choices:       Whether to install (present), or remove (absent) a package.
   state            • present      The upgrade (latest) operation will update/install the package to the latest version available.
   - / required     • absent       Note: The module has a limitation that (latest) only works for one package, not lists of them.
                    • latest
   update_catalog   Choices:
   boolean          • no ←         If you want to refresh your catalog from the mirror, set this to (yes).
                    • yes

EXAMPLES                   *pkgutil-examples*

 # Install a package
 - pkgutil:
     name: CSWcommon
     state: present

 # Install a package from a specific repository
 - pkgutil:
     name: CSWnrpe
     site: 'ftp://myinternal.repo/opencsw/kiel'
     state: latest
