SYNOPSIS                   *vca_nat*

     • Adds or removes nat rules from a gateway in a vca environment

PARAMETERS                   *vca_nat-parameters*

     Parameter    Choices/Defaults                                                                              Comments
   api_version    Default:         The API version to be used with the vca.
   string         "5.7"
   gateway_name   Default:         The name of the gateway of the vdc where the rule should be added.
   string         "gateway"
   host                            The authentication host to be used when service type is vcd.
   string
   instance_id                     The instance ID in a vchs environment to be used for creating the vapp.
   string
   nat_rules      Default:         A list of rules to be added to the gateway, Please see examples on valid entries
   - / required   "no"
   org                             The org to login to for creating vapp.
   string                          This option is required when the service_type is vdc.
   password                        The vca password, if not set the environment variable VCA_PASS is checked for the password.
   string                          aliases: pass, passwd
   purge_rules      Choices:
   boolean          • no ←         If set to true, it will delete all rules in the gateway that are not given as parameter to this module.
                    • yes
                    Choices:
   service_type     • vca ←        The type of service we are authenticating against.
   string           • vcd
                    • vchs
   state            Choices:
   string           • absent       Whether the object should be added or removed.
                    • present ←
   username                        The vca username or email address, if not set the environment variable VCA_USER is checked for the username.
   string                          aliases: user
   validate_certs   Choices:       If the certificates of the authentication is to be verified.
   boolean          • no           aliases: verify_certs
                    • yes ←
   vdc_name                        The name of the vdc where the gateway is located.
   string

EXAMPLES                   *vca_nat-examples*

 #An example for a source nat

 - hosts: localhost
   connection: local
   tasks:
    - vca_nat:
        instance_id: 'b15ff1e5-1024-4f55-889f-ea0209726282'
        vdc_name: 'benz_ansible'
        state: 'present'
        nat_rules:
          - rule_type: SNAT
            original_ip: 192.0.2.42
            translated_ip: 203.0.113.23

 #example for a DNAT
 - hosts: localhost
   connection: local
   tasks:
    - vca_nat:
        instance_id: 'b15ff1e5-1024-4f55-889f-ea0209726282'
        vdc_name: 'benz_ansible'
        state: 'present'
        nat_rules:
          - rule_type: DNAT
            original_ip: 203.0.113.23
            original_port: 22
            translated_ip: 192.0.2.42
            translated_port: 22
