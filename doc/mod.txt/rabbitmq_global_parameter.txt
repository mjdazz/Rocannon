SYNOPSIS                   *rabbitmq_global_parameter*

     • Manage dynamic, cluster-wide global parameters for RabbitMQ

PARAMETERS                   *rabbitmq_global_parameter-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   name         Default:         Name of the global parameter being set
   - / required null
   node         Default:         erlang node name of the rabbit we wish to configure
   -            "rabbit"
   state          Choices:
   -              • present ←    Specify if user is to be added or removed
                  • absent
   value        Default:         Value of the global parameter, as a JSON term
   -            null

EXAMPLES                   *rabbitmq_global_parameter-examples*

 # Set the global parameter 'cluster_name' to a value of 'mq-cluster' (in quotes)
 - rabbitmq_global_parameter:
     name: cluster_name
     value: "{{ 'mq-cluster' | to_json }}"
     state: present

RETURN VALUES                   *rabbitmq_global_parameter-return values*

   Common return values are documented here, the following are the fields unique to this module:

    Key   Returned                                                                                    Description
   name            name of the global parameter being set
   string success  Sample:
                   cluster_name
   value           value of the global parameter, as a JSON term
   string changed  Sample:
                   the-cluster-name
