SYNOPSIS                   *fortios_firewall_auth_portal*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify firewall feature and auth_portal category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_firewall_auth_portal-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_firewall_auth_portal-parameters*

          Parameter        Choices/Defaults                                                                         Comments
   firewall_auth_portal    Default:         Configure firewall authentication portals.
   dictionary              null
    groups                                  Firewall user groups permitted to authenticate through this portal. Separate group names with spaces.
    list
        name                                Group name. Source user.group.name.
        string / required
    identity_based_route                    Name of the identity-based route that applies to this portal. Source firewall.identity-based-route.name.
    string
    portal_addr                             Address (or FQDN) of the authentication portal.
    string
    portal_addr6                            IPv6 address (or FQDN) of authentication portal.
    string
   host                                     FortiOS or FortiGate IP address.
   string
   https                     Choices:
   boolean                   • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                             • yes ←
   password                Default:         FortiOS or FortiGate password.
   string                  ""
   ssl_verify                Choices:
   boolean                   • no           Ensures FortiGate certificate must be verified by a proper CA.
   added in 2.9              • yes ←
   username                                 FortiOS or FortiGate username.
   string
   vdom                    Default:         Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different unit.
   string                  "root"

NOTES                   *fortios_firewall_auth_portal-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_firewall_auth_portal-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure firewall authentication portals.
     fortios_firewall_auth_portal:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       firewall_auth_portal:
         groups:
          -
             name: "default_name_4 (source user.group.name)"
         identity_based_route: "<your_own_value> (source firewall.identity-based-route.name)"
         portal_addr: "<your_own_value>"
         portal_addr6: "<your_own_value>"
