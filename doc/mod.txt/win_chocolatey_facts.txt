SYNOPSIS                   *win_chocolatey_facts*

     • This module shows information from Chocolatey, such as installed packages, configuration, feature and sources.

NOTES                   *win_chocolatey_facts-notes*

   Note

     • Chocolatey must be installed beforehand, use win_chocolatey to do this.

SEE ALSO                   *win_chocolatey_facts-see also*

   See also

   win_chocolatey – Manage packages using chocolatey
   The official documentation on the win_chocolatey module.

   win_chocolatey_config – Manages Chocolatey config settings
   The official documentation on the win_chocolatey_config module.

   win_chocolatey_feature – Manages Chocolatey features
   The official documentation on the win_chocolatey_feature module.

   The official documentation on the win_chocolatey_source module.

EXAMPLES                   *win_chocolatey_facts-examples*

 - name: Gather facts from chocolatey
   win_chocolatey_facts:

 - name: Displays the Configuration
   debug:
     var: ansible_chocolatey.config

 - name: Displays the Feature
   debug:
     var: ansible_chocolatey.feature

 - name: Displays the Sources
   debug:
     var: ansible_chocolatey.sources

 - name: Displays the Packages
   debug:
     var: ansible_chocolatey.packages

RETURNED FACTS                   *win_chocolatey_facts-returned facts*

   Facts returned by this module are added/updated in the hostvars host facts and can be referenced by name just like any other host fact. They do not need to be registered in order to use them.

           Fact         Returned                                                                             Description
   ansible_chocolatey   always   Detailed information about the Chocolatey installation
   complex
    config                       Detailed information about stored the configurations
    dictionary          always   Sample:
                                 {'commandExecutionTimeoutSeconds': 2700, 'containsLegacyPackageInstalls': True}
    feature                      Detailed information about enabled and disabled features
    dictionary          always   Sample:
                                 {'allowEmptyCheckums': False, 'autoUninstaller': True, 'failOnAutoUninstaller': False}
    packages            always   List of installed Packages
    complex
     package                     Name of the package
     string             always   Sample:
                                 vscode
     version                     Version of the package
     string             always   Sample:
                                 1.27.2
    sources             always   List of Chocolatey sources
    complex
     admin_only         always   Is the source visible to Administrators only
     boolean
     allow_self_service always   Is the source allowed to be used with self-service
     boolean
     bypass_proxy                Can the source explicitly bypass configured proxies
     boolean            always   Sample:
                                 True
     certificate                 Pth to a PFX certificate for X509 authenticated feeds
     string             always   Sample:
                                 C:\chocolatey\cert.pfx
     disabled           always   Is the source disabled
     boolean
     name                        Name of the source
     string             always   Sample:
                                 chocolatey
     priority           always   The priority order of this source, lower is better, 0 is no priority
     integer
     source                      The source, can be a folder/file or an url
     string             always   Sample:
                                 https://chocolatey.org/api/v2/
     source_username             Username used to access authenticated feeds
     string             always   Sample:
                                 username
