SYNOPSIS                   *fail*

     • This module fails the progress with a custom message.
     • It can be useful for bailing out when a certain condition is met using when.
     • This module is also supported for Windows targets.

PARAMETERS                   *fail-parameters*

   Parameter        Choices/Defaults                                                                                 Comments
   msg       Default:                        The customized message used for failing execution.
   string    "Failed as requested from task" If omitted, fail will simply bail out with a generic message.

NOTES                   *fail-notes*

   Note

     • This module is also supported for Windows targets.

EXAMPLES                   *fail-examples*

 # Example playbook using fail and when together
 - fail:
     msg: The system may not be provisioned according to the CMDB status.
   when: cmdb_status != "to-be-staged"
