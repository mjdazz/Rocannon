SYNOPSIS                   *pn_user*

     • This module can be used to create a user and apply a role, update a user and delete a user.

PARAMETERS                   *pn_user-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   pn_cliswitch                       Target switch to run the CLI on.
   string
   pn_initial_role                    initial role for user.
   string
   pn_name                            username.
   string
   pn_password                        plain text password.
   string
   pn_scope            Choices:
   string              • local        local or fabric.
                       • fabric
                       Choices:
   state               • present      State the action to perform. Use present to create user and absent to delete user update to update user.
   string / required   • absent
                       • update

EXAMPLES                   *pn_user-examples*

 - name: Create user
   pn_user:
     pn_cliswitch: "sw01"
     state: "present"
     pn_scope: "fabric"
     pn_password: "foo123"
     pn_name: "foo"

 - name: Delete user
   pn_user:
     pn_cliswitch: "sw01"
     state: "absent"
     pn_name: "foo"

 - name: Modify user
   pn_user:
     pn_cliswitch: "sw01"
     state: "update"
     pn_password: "test1234"
     pn_name: "foo"

RETURN VALUES                   *pn_user-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   indicates whether the CLI caused changes on the target.
   boolean
   command always   the CLI command run on the target node.
   string
   stderr  on error set of error responses from the user command.
   list
   stdout  always   set of responses from the user command.
   list
