SYNOPSIS                   *pn_admin_syslog*

     • This module can be used to create the scope and other parameters of syslog event collection.
     • This module can be used to modify parameters of syslog event collection.
     • This module can be used to delete the scope and other parameters of syslog event collection.

PARAMETERS                   *pn_admin_syslog-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   pn_cliswitch                       Target switch to run the CLI on.
   string
   pn_host                            Hostname to log system events.
   string
   pn_message_format   Choices:
   string              • structured   message-format for log events - structured or legacy.
                       • legacy
   pn_name                            name of the system log.
   string
   pn_port                            Host port.
   string
   pn_scope            Choices:
   string              • local        Scope of the system log.
                       • fabric
   pn_transport        Choices:
   string              • tcp-tls      Transport for log events - tcp/tls or udp.
                       • udp ←
                       Choices:
   state               • present      State the action to perform. Use present to create admin-syslog and absent to delete admin-syslog update to modify the admin-syslog.
   string / required   • absent
                       • update

EXAMPLES                   *pn_admin_syslog-examples*

 - name: admin-syslog functionality
   pn_admin_syslog:
     pn_cliswitch: "sw01"
     state: "absent"
     pn_name: "foo"
     pn_scope: "local"

 - name: admin-syslog functionality
   pn_admin_syslog:
     pn_cliswitch: "sw01"
     state: "present"
     pn_name: "foo"
     pn_scope: "local"
     pn_host: "166.68.224.46"
     pn_message_format: "structured"

 - name: admin-syslog functionality
   pn_admin_syslog:
     pn_cliswitch: "sw01"
     state: "update"
     pn_name: "foo"
     pn_host: "166.68.224.10"

RETURN VALUES                   *pn_admin_syslog-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   indicates whether the CLI caused changes on the target.
   boolean
   command always   the CLI command run on the target node.
   string
   stderr  on error set of error responses from the admin-syslog command.
   list
   stdout  always   set of responses from the admin-syslog command.
   list
