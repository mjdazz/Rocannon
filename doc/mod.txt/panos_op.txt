SYNOPSIS                   *panos_op-synopsis*

     • This module will allow user to pass and execute any supported OP command on the PANW device.

REQUIREMENTS                   *panos_op-requirements*

   The below requirements are needed on the host that executes this module.

     • pan-python can be obtained from PyPI https://pypi.org/project/pan-python/
     • pandevice can be obtained from PyPI https://pypi.org/project/pandevice/

PARAMETERS                   *panos_op-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   api_key                       API key that can be used instead of username/password credentials.
   -
   cmd                           The OP command to be performed.
   - / required
   ip_address                    IP address (or hostname) of PAN-OS device or Panorama management console being configured.
   - / required
   password                      Password credentials to use for authentication.
   - / required
   username     Default:         Username credentials to use for authentication.
   -            "admin"

NOTES                   *panos_op-notes*

   Note

     • Checkmode is NOT supported.
     • Panorama is NOT supported.

EXAMPLES                   *panos_op-examples*

 - name: show list of all interfaces
   panos_op:
     ip_address: '{{ ip_address }}'
     username: '{{ username }}'
     password: '{{ password }}'
     cmd: 'show interfaces all'

 - name: show system info
   panos_op:
     ip_address: '{{ ip_address }}'
     username: '{{ username }}'
     password: '{{ password }}'
     cmd: 'show system info'

RETURN VALUES                   *panos_op-return values*

   Common return values are documented here, the following are the fields unique to this module:

      Key     Returned                                                                                  Description
   stdout              output of the given OP command as JSON formatted string
   string     success  Sample:
                       {system: {app-release-date: 2017/05/01 15:09:12}}
   stdout_xml          output of the given OP command as JSON formatted string
   string     success  Sample:
                       <response status=success><result><system><hostname>fw2</hostname>
