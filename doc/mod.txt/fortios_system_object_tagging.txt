SYNOPSIS                   *fortios_system_object_tagging*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify system feature and object_tagging category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_system_object_tagging-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

EXAMPLES                   *fortios_system_object_tagging-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure object tagging.
     fortios_system_object_tagging:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       state: "present"
       system_object_tagging:
         address: "disable"
         category: "<your_own_value>"
         color: "5"
         device: "disable"
         interface: "disable"
         multiple: "enable"
         tags:
          -
             name: "default_name_10"
