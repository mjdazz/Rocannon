SYNOPSIS                   *bigmon_policy*

     • Create and remove a bigmon out-of-band policy.

PARAMETERS                   *bigmon_policy-parameters*

         Parameter            Choices/Defaults                                                                           Comments
   access_token                                      Bigmon access token. If this isn't set, the environment variable BIGSWITCH_ACCESS_TOKEN is used.
   -
                           Choices:
   action                  • forward ←               Forward matching packets to delivery interfaces, Drop is for measure rate of matching packets, but do not forward to delivery interfaces,
   -                       • drop                    capture packets and write to a PCAP file, or enable NetFlow generation.
                           • flow-gen
   controller                                        The controller address.
   - / required
   delivery_packet_count Default:                    Run policy until delivery_packet_count packets are delivered.
   -                     0
   duration              Default:                    Run policy for duration duration or until delivery_packet_count packets are delivered, whichever comes first.
   -                     0
   name                                              The name of the policy.
   - / required
   policy_description                                Description of policy.
   -
   priority              Default:                    A priority associated with this policy. The higher priority policy takes precedence over a lower priority.
   -                     100
   start_time            Default:                    Date the policy becomes active
   -                     "ansible_date_time.iso8601"
   state                   Choices:
   -                       • present ←               Whether the policy should be present or absent.
                           • absent
   validate_certs          Choices:
   boolean                 • no                      If false, SSL certificates will not be validated. This should only be used on personally controlled devices using self-signed certificates.
                           • yes ←

EXAMPLES                   *bigmon_policy-examples*

 - name: policy to aggregate filter and deliver data center (DC) 1 traffic
   bigmon_policy:
     name: policy1
     policy_description: DC 1 traffic policy
     action: drop
     controller: '{{ inventory_hostname }}'
     state: present
     validate_certs: false
