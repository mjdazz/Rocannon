SYNOPSIS                   *monit*

     • Manage the state of a program monitored via Monit

PARAMETERS                   *monit-parameters*

   Parameter Choices/Defaults                                                                                Comments
   name
   - /                        The name of the monit program/process to manage
   required
               Choices:
               • present
   state       • started
   - /         • stopped      The state of service
   required    • restarted
               • monitored
               • unmonitored
               • reloaded
   timeout   Default:         If there are pending actions for the service monitored by monit, then Ansible will check for up to this many seconds to verify the requested action has been performed.
   -         300              Ansible will sleep for five seconds between each check.

EXAMPLES                   *monit-examples*

 # Manage the state of program "httpd" to be in "started" state.
 - monit:
     name: httpd
     state: started
