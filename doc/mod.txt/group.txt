SYNOPSIS                   *group*

     • Manage presence of groups on a host.
     • For Windows targets, use the win_group module instead.

REQUIREMENTS                   *group-requirements*

   The below requirements are needed on the host that executes this module.

     • groupadd
     • groupdel
     • groupmod

PARAMETERS                   *group-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   gid                                Optional GID to set for the group.
   integer
   local               Choices:       Forces the use of "local" command alternatives on platforms that implement it.
   boolean             • no ←         This is useful in environments that use centralized authentication when you want to manipulate the local groups. (e.g. it uses lgroupadd instead of groupadd).
   added in 2.6        • yes          This requires that these commands exist on the targeted host, otherwise it will be a fatal error.
   name                               Name of the group to manage.
   string / required
   non_unique          Choices:       This option allows to change the group ID to a non-unique value. Requires gid.
   boolean             • no ←         Not supported on macOS or BusyBox distributions.
   added in 2.8        • yes
   state               Choices:
   string              • absent       Whether the group should be present or not on the remote host.
                       • present ←
   system              Choices:
   boolean             • no ←         If yes, indicates that the group created is a system group.
                       • yes

EXAMPLES                   *group-examples*

 - name: Ensure group "somegroup" exists
   group:
     name: somegroup
     state: present
