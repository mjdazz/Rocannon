SYNOPSIS                   *rhn_register*

     • Manage registration to the Red Hat Network.

REQUIREMENTS                   *rhn_register-requirements*

   The below requirements are needed on the host that executes this module.

     • rhnreg_ks
     • either libxml2 or lxml

PARAMETERS                   *rhn_register-parameters*

     Parameter   Choices/Defaults                                                                              Comments
   activationkey                  Supply an activation key for use with registration.
   string
   ca_cert                        Supply a custom ssl CA certificate file for use with registration.
   path                           aliases: sslcacert
   channels      Default:         Optionally specify a list of channels to subscribe to upon successful registration.
   list          []
   enable_eus      Choices:
   boolean         • no ←         If no, extended update support will be requested.
                   • yes
   nopackages      Choices:
   boolean         • no ←         If yes, the registered node will not upload its installed packages information to Satellite server.
   added in 2.5    • yes
   password                       Red Hat Network password.
   string
   profilename                    Supply an profilename for use with registration.
   string
   server_url                     Specify an alternative Red Hat Network server URL.
   string                         The default is the current value of serverURL from /etc/sysconfig/rhn/up2date.
   state           Choices:
   string          • absent       Whether to register (present), or unregister (absent) a system.
                   • present ←
   systemorgid                    Supply an organizational id for use with registration.
   string
   username                       Red Hat Network username.
   string

NOTES                   *rhn_register-notes*

   Note

     • This is for older Red Hat products. You probably want the redhat_subscription module instead.
     • In order to register a system, rhnreg_ks requires either a username and password, or an activationkey.

EXAMPLES                   *rhn_register-examples*

 - name: Unregister system from RHN
   rhn_register:
     state: absent
     username: joe_user
     password: somepass

 - name: Register as user with password and auto-subscribe to available content
   rhn_register:
     state: present
     username: joe_user
     password: somepass

 - name: Register with activationkey and enable extended update support
   rhn_register:
     state: present
     activationkey: 1-222333444
     enable_eus: yes

 - name: Register with activationkey and set a profilename which may differ from the hostname
   rhn_register:
     state: present
     activationkey: 1-222333444
     profilename: host.example.com.custom

 - name: Register as user with password against a satellite server
   rhn_register:
     state: present
     username: joe_user
     password: somepass
     server_url: https://xmlrpc.my.satellite/XMLRPC

 - name: Register as user with password and enable channels
   rhn_register:
     state: present
     username: joe_user
     password: somepass
     channels: rhel-x86_64-server-6-foo-1,rhel-x86_64-server-6-bar-1
