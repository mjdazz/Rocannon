SYNOPSIS                   *openvswitch_db*

     • Set column values in record in database table.

REQUIREMENTS                   *openvswitch_db-requirements*

   The below requirements are needed on the host that executes this module.

     • ovs-vsctl >= 2.3.3

PARAMETERS                   *openvswitch_db-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   col                           Identifies the column in the record.
   - / required
   key                           Identifies the key in the record column, when the column is a map type.
   -
   record                        Identifies the record in the table.
   - / required
   state          Choices:
   -              • present ←    Configures the state of the key. When set to present, the key and value pair will be set on the record and when set to absent the key will not be set.
   added in 2.4   • absent
   table                         Identifies the table in the database.
   - / required
   timeout      Default:         How long to wait for ovs-vswitchd to respond
   -            5
   value                         Expected value for the table, record, column and key.
   - / required

EXAMPLES                   *openvswitch_db-examples*

 # Increase the maximum idle time to 50 seconds before pruning unused kernel
 # rules.
 - openvswitch_db:
     table: open_vswitch
     record: .
     col: other_config
     key: max-idle
     value: 50000

 # Disable in band copy
 - openvswitch_db:
     table: Bridge
     record: br-int
     col: other_config
     key: disable-in-band
     value: true

 # Remove in band key
 - openvswitch_db:
     state: present
     table: Bridge
     record: br-int
     col: other_config
     key: disable-in-band

 # Mark port with tag 10
 - openvswitch_db:
     table: Port
     record: port0
     col: tag
     value: 10
