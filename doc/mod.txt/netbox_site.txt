SYNOPSIS                   *netbox_site*

     • Creates or removes sites from Netbox

REQUIREMENTS                   *netbox_site-requirements*

   The below requirements are needed on the host that executes this module.

     • pynetbox

PARAMETERS                   *netbox_site-parameters*

       Parameter      Choices/Defaults                                                                            Comments
   data                                Defines the site configuration
   - / required
    asn                                The ASN associated with the site
    integer
    comments                           Comments for the site. This can be markdown syntax
    string
    contact_email                      Contact email for site
    string
    contact_name                       Name of contact for site
    string
    contact_phone                      Contact phone number for site
    string
    custom_fields                      must exist in Netbox
    dictionary
    description                        The description of the prefix
    string
    facility                           Data center provider or facility, ex. Equinix NY7
    string
    latitude                           Latitude in decimal format
    integer
    longitude                          Longitude in decimal format
    integer
    name                               Name of the site to be created
    string / required
    physical_address                   Physical address of site
    string
    region                             The region that the site should be associated with
    string
    shipping_address                   Shipping address of site
    string
                        Choices:
    status              • Active       Status of the site
    string              • Planned
                        • Retired
    tags                               Any tags that the prefix may need to be associated with
    list
    tenant                             The tenant the site will be assigned to
    string
    time_zone                          Timezone associated with the site, ex. America/Denver
    string
   netbox_token                        The token created within Netbox to authorize API access
   string / required
   netbox_url                          URL of the Netbox instance resolvable by Ansible control host
   string / required
   state                Choices:
   string               • absent       Use present or absent for adding or removing.
                        • present ←
   validate_certs       Choices:
   boolean              • no           If no, SSL certificates will not be validated. This should only be used on personally controlled sites using self-signed certificates.
                        • yes ←

NOTES                   *netbox_site-notes*

   Note

     • Tags should be defined as a YAML list
     • This should be ran with connection local and hosts localhost

EXAMPLES                   *netbox_site-examples*

 - name: "Test Netbox site module"
   connection: local
   hosts: localhost
   gather_facts: False
   tasks:
     - name: Create site within Netbox with only required information
       netbox_site:
         netbox_url: http://netbox.local
         netbox_token: thisIsMyToken
         data:
           name: Test - Colorado
         state: present

     - name: Delete site within netbox
       netbox_site:
         netbox_url: http://netbox.local
         netbox_token: thisIsMyToken
         data:
           name: Test - Colorado
         state: absent

     - name: Create site with all parameters
       netbox_site:
         netbox_url: http://netbox.local
         netbox_token: thisIsMyToken
         data:
           name: Test - California
           status: Planned
           region: Test Region
           tenant: Test Tenant
           facility: EquinoxCA7
           asn: 65001
           time_zone: America/Los Angeles
           description: This is a test description
           physical_address: Hollywood, CA, 90210
           shipping_address: Hollywood, CA, 90210
           latitude: 10.100000
           longitude: 12.200000
           contact_name: Jenny
           contact_phone: 867-5309
           contact_email: [email protected]
           comments: ### Placeholder
         state: present
