DEPRECATED                   *na_cdot_qtree*

                            Removed in Ansible:
                  version: 2.11
        Why:      Updated modules released with increased functionality
    Alternative:  Use na_ontap_qtree instead.

SYNOPSIS                   *na_cdot_qtree-synopsis*

     • Create or destroy Qtrees.

REQUIREMENTS                   *na_cdot_qtree-requirements*

   The below requirements are needed on the host that executes this module.

     • A physical or virtual clustered Data ONTAP system. The modules were developed with Clustered Data ONTAP 8.3
     • Ansible 2.2
     • netapp-lib (2015.9.25). Install using ‘pip install netapp-lib’

PARAMETERS                   *na_cdot_qtree-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   flexvol_name                  The name of the FlexVol the Qtree should exist on. Required when state=present.
   -
   hostname                      The hostname or IP address of the ONTAP instance.
   - / required
   name                          The name of the Qtree to manage.
   - / required
   password                      Password for the specified user.
   - / required                  aliases: pass
   state          Choices:
   - / required   • present      Whether the specified Qtree should exist or not.
                  • absent
   username                      This can be a Cluster-scoped or SVM-scoped account, depending on whether a Cluster-level or SVM-level API is required. For more information, please read the
   - / required                  documentation https://mysupport.netapp.com/NOW/download/software/nmsdk/9.4/.
                                 aliases: user
   vserver                       The name of the vserver to use.
   - / required

NOTES                   *na_cdot_qtree-notes*

   Note

     • The modules prefixed with na\_cdot are built to support the ONTAP storage platform.

EXAMPLES                   *na_cdot_qtree-examples*

 - name: Create QTree
   na_cdot_qtree:
     state: present
     name: ansibleQTree
     flexvol_name: ansibleVolume
     vserver: ansibleVServer
     hostname: "{{ netapp_hostname }}"
     username: "{{ netapp_username }}"
     password: "{{ netapp_password }}"

 - name: Rename QTree
   na_cdot_qtree:
     state: present
     name: ansibleQTree
     flexvol_name: ansibleVolume
     vserver: ansibleVServer
     hostname: "{{ netapp_hostname }}"
     username: "{{ netapp_username }}"
     password: "{{ netapp_password }}"
