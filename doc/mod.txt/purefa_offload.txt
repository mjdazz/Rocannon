SYNOPSIS                   *purefa_offload*

     • Create, modify and delete NFS or S3 offload targets.
     • Only supported on Purity v5.2.0 or higher.
     • You must have a correctly configured offload network for offload to work.

REQUIREMENTS                   *purefa_offload-requirements*

   The below requirements are needed on the host that executes this module.

     • python >= 2.7
     • purestorage

PARAMETERS                   *purefa_offload-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   access_key                         Access Key ID of the S3 target
   string
   address                            The IP or FQDN address of the NFS server
   string
   api_token                          FlashArray API token for admin privileged user.
   string / required
   bucket                             Name of the bucket for the S3 target
   string
   fa_url                             FlashArray management IPv4 address or Hostname.
   string / required
   initialize          Choices:
   boolean             • no           Define whether to initialize the S3 bucket
                       • yes ←
   name                               The name of the offload target
   string / required
   options           Default:         Additional mount options for the NFS share
   string            ""               Supported mount options include port, rsize, wsize, nfsvers, and tcp or udp
   protocol            Choices:
   string              • nfs ←        Define which protocol the offload engine uses
                       • s3
   secret                             Secret Access Key for the S3 target
   string
   share                              NFS export on the NFS server
   string
   state               Choices:
   string              • absent       Define state of offload
                       • present ←

NOTES                   *purefa_offload-notes*

   Note

     • This module requires the purestorage Python library
     • You must set PUREFA_URL and PUREFA_API environment variables if fa_url and api_token arguments are not passed to the module directly

EXAMPLES                   *purefa_offload-examples*

 - name: Create NFS offload target
   purefa_offload:
     name: nfs-offload
     protocol: nfs
     address: 10.21.200.4
     share: "/offload_target"
     fa_url: 10.10.10.2
     api_token: e31060a7-21fc-e277-6240-25983c6c4592

 - name: Create S3 offload target
   purefa_offload:
     name: s3-offload
     protocol: s3
     access_key: "3794fb12c6204e19195f"
     bucket: offload-bucket
     secret: "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
     fa_url: 10.10.10.2
     api_token: e31060a7-21fc-e277-6240-25983c6c4592

 - name: Delete offload target
   purefa_offload:
     name: nfs-offload
     protocol: nfs
     state: absent
     fa_url: 10.10.10.2
     api_token: e31060a7-21fc-e277-6240-25983c6c4592
