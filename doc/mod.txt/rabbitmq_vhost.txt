SYNOPSIS                   *rabbitmq_vhost*

     • Manage the state of a virtual host in RabbitMQ

PARAMETERS                   *rabbitmq_vhost-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   name                          The name of the vhost to manage
   - / required                  aliases: vhost
   node         Default:         erlang node name of the rabbit we wish to configure
   -            "rabbit"
   state          Choices:
   -              • present ←    The state of vhost
                  • absent
   tracing        Choices:       Enable/disable tracing for a vhost
   boolean        • no ←         aliases: trace
                  • yes

EXAMPLES                   *rabbitmq_vhost-examples*

 # Ensure that the vhost /test exists.
 - rabbitmq_vhost:
     name: /test
     state: present
