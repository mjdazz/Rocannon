SYNOPSIS                   *fortios_web_proxy_forward_server*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify web_proxy feature and forward_server category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_web_proxy_forward_server-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

EXAMPLES                   *fortios_web_proxy_forward_server-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure forward-server addresses.
     fortios_web_proxy_forward_server:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       state: "present"
       web_proxy_forward_server:
         addr_type: "ip"
         comment: "Comment."
         fqdn: "<your_own_value>"
         healthcheck: "disable"
         ip: "<your_own_value>"
         monitor: "<your_own_value>"
         name: "default_name_9"
         port: "10"
         server_down_option: "block"
