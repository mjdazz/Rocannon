SYNOPSIS                   *fortios_firewall_identity_based_route*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify firewall feature and identity_based_route category. Examples include all
       parameters and values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_firewall_identity_based_route-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

EXAMPLES                   *fortios_firewall_identity_based_route-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure identity based routing.
     fortios_firewall_identity_based_route:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       state: "present"
       firewall_identity_based_route:
         comments: "<your_own_value>"
         name: "default_name_4"
         rule:
          -
             device: "<your_own_value> (source system.interface.name)"
             gateway: "<your_own_value>"
             groups:
              -
                 name: "default_name_9 (source user.group.name)"
             id:  "10"
