SYNOPSIS                   *bzr*

     • Manage bzr branches to deploy files or software.

PARAMETERS                   *bzr-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   dest                          Absolute path of where the branch should be cloned to.
   - / required
   executable                    Path to bzr executable to use. If not supplied, the normal mechanism for resolving binary paths will be used.
   -
   force          Choices:
   boolean        • no ←         If yes, any modified files in the working tree will be discarded. Before 1.9 the default value was yes.
                  • yes
   name                          SSH or HTTP protocol address of the parent branch.
   - / required                  aliases: parent
   version      Default:         What version of the branch to clone. This can be the bzr revno or revid.
   -            "head"

EXAMPLES                   *bzr-examples*

 # Example bzr checkout from Ansible Playbooks
 - bzr:
     name: bzr+ssh://foosball.example.org/path/to/branch
     dest: /srv/checkout
     version: 22
