SYNOPSIS                   *fortios_system_csf*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify system feature and csf category. Examples include all parameters and values need
       to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_system_csf-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

NOTES                   *fortios_system_csf-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_system_csf-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Add this FortiGate to a Security Fabric or set up a new Security Fabric on this FortiGate.
     fortios_system_csf:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       system_csf:
         configuration_sync: "default"
         fabric_device:
          -
             device_ip: "<your_own_value>"
             device_type: "fortimail"
             login: "<your_own_value>"
             name: "default_name_8"
             password: "<your_own_value>"
         fixed_key: "<your_own_value>"
         group_name: "<your_own_value>"
         group_password: "<your_own_value>"
         management_ip: "<your_own_value>"
         management_port: "14"
         status: "enable"
         trusted_list:
          -
             action: "accept"
             downstream_authorization: "enable"
             ha_members: "<your_own_value>"
             serial: "<your_own_value>"
         upstream_ip: "<your_own_value>"
         upstream_port: "22"
