SYNOPSIS                   *honeybadger_deployment*

     • Notify Honeybadger.io about app deployments (see http://docs.honeybadger.io/article/188-deployment-tracking)

PARAMETERS                   *honeybadger_deployment-parameters*

     Parameter               Choices/Defaults                                                                              Comments
   environment                                            The environment name, typically 'production', 'staging', etc.
   - / required
   repo                                                   URL of the project repository
   -
   revision                                               A hash, number, tag, or other identifier showing what revision was deployed
   -
   token                                                  API token.
   - / required
   url            Default:                                Optional URL to submit the notification to.
   -              "https://api.honeybadger.io/v1/deploys"
   user                                                   The username of the person doing the deployment
   -
   validate_certs   Choices:                              If no, SSL certificates for the target url will not be validated. This should only be used on personally controlled sites using self-signed
   boolean          • no                                  certificates.
                    • yes ←

EXAMPLES                   *honeybadger_deployment-examples*

 - honeybadger_deployment:
     token: AAAAAA
     environment: staging
     user: ansible
     revision: b6826b8
     repo: '[email protected]:user/repo.git'
