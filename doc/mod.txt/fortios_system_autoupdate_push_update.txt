SYNOPSIS                   *fortios_system_autoupdate_push_update*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify system_autoupdate feature and push_update category. Examples include all
       parameters and values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_system_autoupdate_push_update-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_system_autoupdate_push_update-parameters*

             Parameter           Choices/Defaults                                                                      Comments
   host                                           FortiOS or FortiGate IP address.
   string
   https                           Choices:
   boolean                         • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                                   • yes ←
   password                      Default:         FortiOS or FortiGate password.
   string                        ""
   ssl_verify                      Choices:
   boolean                         • no           Ensures FortiGate certificate must be verified by a proper CA.
                                   • yes ←
   system_autoupdate_push_update Default:         Configure push updates.
   dictionary                    null
               address                            Push update override server.
               string
               override            Choices:
               string              • enable       Enable/disable push update override server.
                                   • disable
               port                               Push update override port. (Do not overlap with other service ports)
               integer
               status              Choices:
               string              • enable       Enable/disable push updates.
                                   • disable
   username                                       FortiOS or FortiGate username.
   string
   vdom                          Default:         Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different unit.
   string                        "root"

NOTES                   *fortios_system_autoupdate_push_update-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_system_autoupdate_push_update-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure push updates.
     fortios_system_autoupdate_push_update:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       system_autoupdate_push_update:
         address: "<your_own_value>"
         override: "enable"
         port: "5"
         status: "enable"
