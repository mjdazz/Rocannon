SYNOPSIS                   *rabbitmq_policy*

     • Manage the state of a policy in RabbitMQ.

PARAMETERS                   *rabbitmq_policy-parameters*

    Parameter   Choices/Defaults                                                                               Comments
                  Choices:
   apply_to       • all ←        What the policy applies to. Requires RabbitMQ 3.2.0 or later.
   -              • exchanges
                  • queues
   name                          The name of the policy to manage.
   - / required
   node         Default:         Erlang node name of the rabbit we wish to configure.
   -            "rabbit"
   pattern      Default:         A regex of queues to apply the policy to. Required when state=present. This option is no longer required as of Ansible 2.9.
   -            null
   priority     Default:         The priority of the policy.
   -            0
   state          Choices:
   -              • present ←    The state of the policy.
                  • absent
   tags         Default:         A dict or string describing the policy. Required when state=present. This option is no longer required as of Ansible 2.9.
   -            null
   vhost        Default:         The name of the vhost to apply to.
   -            "/"

EXAMPLES                   *rabbitmq_policy-examples*

 - name: ensure the default vhost contains the HA policy via a dict
   rabbitmq_policy:
     name: HA
     pattern: ."
   args:
     tags:
       ha-mode: all

 - name: ensure the default vhost contains the HA policy
   rabbitmq_policy:
     name: HA
     pattern: ."
     tags:
       ha-mode: all
