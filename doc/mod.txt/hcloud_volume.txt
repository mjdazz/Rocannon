SYNOPSIS                   *hcloud_volume*

     • Create, update and attach/detach block volumes on the Hetzner Cloud.

REQUIREMENTS                   *hcloud_volume-requirements*

   The below requirements are needed on the host that executes this module.

     • hcloud-python >= 1.0.0

PARAMETERS                   *hcloud_volume-parameters*

       Parameter            Choices/Defaults                                                                            Comments
   api_token                                        This is the API Token for the Hetzner Cloud.
   string / required
   automount           Choices:
   boolean             • no                         Automatically mount the Volume.
                       • yes
   endpoint          Default:                       This is the API Endpoint for the Hetzner Cloud.
   string            "https://api.hetzner.cloud/v1"
   format              Choices:                     Automatically Format the volume on creation
   string              • xfs                        Can only be used in case the Volume does not exists.
                       • ext4
   id                                               The ID of the Hetzner Cloud Block Volume to manage.
   integer                                          Only required if no volume name is given
   labels                                           User-defined key-value pairs.
   dictionary
   location                                         Location of the Hetzner Cloud Volume.
   string                                           Required if no server is given and Volume does not exists.
   name                                             The Name of the Hetzner Cloud Block Volume to manage.
   string                                           Only required if no volume id is given or a volume does not exists.
   server                                           Server Name the Volume should be assigned to.
   string                                           Required if no location is given and Volume does not exists.
   size                                             The size of the Block Volume in GB.
   integer                                          Required if volume does not yet exists.
   state               Choices:
   string              • absent                     State of the volume.
                       • present ←

EXAMPLES                   *hcloud_volume-examples*

 - name: Create a volume
   hcloud_volume:
     name: my-volume
     location: fsn1
     size: 100
     state: present
 - name: Create a volume and format it with ext4
   hcloud_volume:
     name: my-volume
     location: fsn
     format: ext4
     size: 100
     state: present
 - name: Mount a existing volume and automount
   hcloud_volume:
     name: my-volume
     server: my-server
     automount: yes
     state: present
 - name: Mount a existing volume and automount
   hcloud_volume:
     name: my-volume
     server: my-server
     automount: yes
     state: present
 - name: Ensure the volume is absent (remove if needed)
   hcloud_volume:
     name: my-volume
     state: absent
