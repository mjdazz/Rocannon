SYNOPSIS                   *rabbitmq_vhost_limits*

     • This module sets/clears certain limits on a virtual host.
     • The configurable limits are max_connections and max-queues.

PARAMETERS                   *rabbitmq_vhost_limits-parameters*

      Parameter    Choices/Defaults                                                                             Comments
   max_connections Default:         Max number of concurrent client connections.
   -               -1               Negative value means "no limit".
                                    Ignored when the state is absent.
   max_queues      Default:         Max number of queues.
   -               -1               Negative value means "no limit".
                                    Ignored when the state is absent.
   node                             Name of the RabbitMQ Erlang node to manage.
   -
   state             Choices:       Specify whether the limits are to be set or cleared.
   -                 • present ←    If set to absent, the limits of both max_connections and max-queues will be cleared.
                     • absent
   vhost           Default:         Name of the virtual host to manage.
   -               "/"

EXAMPLES                   *rabbitmq_vhost_limits-examples*

 # Limit both of the max number of connections and queues on the vhost '/'.
 - rabbitmq_vhost_limits:
     vhost: /
     max_connections: 64
     max_queues: 256
     state: present

 # Limit the max number of connections on the vhost '/'.
 # This task implicitly clears the max number of queues limit using default value: -1.
 - rabbitmq_vhost_limits:
     vhost: /
     max_connections: 64
     state: present

 # Clear the limits on the vhost '/'.
 - rabbitmq_vhost_limits:
     vhost: /
     state: absent
