SYNOPSIS                   *win_pester*

     • Run Pester tests on Windows hosts.
     • Test files have to be available on the remote host.

REQUIREMENTS                   *win_pester-requirements*

   The below requirements are needed on the host that executes this module.

     • Pester

PARAMETERS                   *win_pester-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   path                               Path to a pester test file or a folder where tests can be found.
   string / required                  If the path is a folder, the module will consider all ps1 files as Pester tests.
   tags                               Runs only tests in Describe blocks with specified Tags values.
   list                               Accepts multiple comma separated tags.
   added in 2.9
   test_parameters
   dictionary                         Allows to specify parameters to the test script.
   added in 2.9
   version                            Minimum version of the pester module that has to be available on the remote host.
   -

EXAMPLES                   *win_pester-examples*

 - name: Get facts
   setup:

 - name: Add Pester module
   action:
     module_name: "{{ 'win_psmodule' if ansible_powershell_version >= 5 else 'win_chocolatey' }}"
     name: Pester
     state: present

 - name: Run the pester test provided in the path parameter.
   win_pester:
     path: C:\Pester

 - name: Run the pester tests only for the tags specified.
   win_pester:
     path: C:\Pester\TestScript.tests
     tags: CI,UnitTests

 # Run pesters tests files that are present in the specified folder
 # ensure that the pester module version available is greater or equal to the version parameter.
 - name: Run the pester test present in a folder and check the Pester module version.
   win_pester:
     path: C:\Pester\test01.test.ps1
     version: 4.1.0

 - name: Run the pester test present in a folder with given script parameters.
   win_pester:
     path: C:\Pester\test04.test.ps1
     test_parameters:
       Process: lsass
       Service: bits

RETURN VALUES                   *win_pester-return values*

   Common return values are documented here, the following are the fields unique to this module:

        Key       Returned                                                                                Description
   output         success  Results of the Pester tests.
   list
   pester_version          Version of the pester module found on the remote host.
   string         always   Sample:
                           4.3.1
