SYNOPSIS                   *riak*

     • This module can be used to join nodes to a cluster, check the status of the cluster.

PARAMETERS                   *riak-parameters*

       Parameter      Choices/Defaults                                                                             Comments
                       Choices:
                       • ping
   command             • kv_test         The command you would like to perform against the cluster.
   -                   • join
                       • plan
                       • commit
   config_dir        Default:            The path to the riak configuration directory
   -                 "/etc/riak"
   http_conn         Default:            The ip address and port that is listening for Riak HTTP queries
   -                 "127.0.0.1:8098"
   target_node       Default:            The target node for certain operations (join, ping)
   -                 "[email protected]"
   validate_certs      Choices:
   boolean             • no              If no, SSL certificates will not be validated. This should only be used on personally controlled sites using self-signed certificates.
                       • yes ←
   wait_for_handoffs                     Number of seconds to wait for handoffs to complete.
   -
   wait_for_ring                         Number of seconds to wait for all nodes to agree on the ring.
   -
   wait_for_service    Choices:          Waits for a riak service to come online before continuing.
   -                   • kv

EXAMPLES                   *riak-examples*

 # Join's a Riak node to another node
 - riak:
     command: join
     target_node: [email protected]

 # Wait for handoffs to finish.  Use with async and poll.
 - riak:
     wait_for_handoffs: yes

 # Wait for riak_kv service to startup
 - riak:
     wait_for_service: kv
