SYNOPSIS                   *netapp_e_storage_system*

     • Manage the arrays accessible via a NetApp Web Services Proxy for NetApp E-series storage arrays.

PARAMETERS                   *netapp_e_storage_system-parameters*

        Parameter       Choices/Defaults                                                                           Comments
   api_password                          The password to authenticate with the SANtricity WebServices Proxy or embedded REST API.
   - / required
   api_url                               The url to the SANtricity WebServices Proxy or embedded REST API.
   - / required
   api_username                          The username to authenticate with the SANtricity WebServices Proxy or embedded REST API.
   - / required
   array_password                        The management password of the array to manage, if set.
   -
   array_wwn                             The WWN of the array to manage. Only necessary if in-band managing multiple arrays on the same agent host. Mutually exclusive of controller_addresses
   -                                     parameter.
   controller_addresses                  The list addresses for the out-of-band management adapter or the agent host. Mutually exclusive of array_wwn parameter.
   - / required
   enable_trace           Choices:
   boolean                • no ←         Enable trace logging for SYMbol calls to the storage system.
                          • yes
   meta_tags                             Optional meta tags to associate to this storage system
   -
   ssid                                  The ID of the array to manage. This value must be unique for each array.
   - / required
   state                  Choices:
   - / required           • present      Whether the specified array should be configured on the Web Services Proxy or not.
                          • absent
   validate_certs         Choices:
   boolean                • no           Should https certificates be validated?
                          • yes ←

EXAMPLES                   *netapp_e_storage_system-examples*

 ---
     - name:  Presence of storage system
       netapp_e_storage_system:
         ssid: "{{ item.key }}"
         state: present
         api_url: "{{ netapp_api_url }}"
         api_username: "{{ netapp_api_username }}"
         api_password: "{{ netapp_api_password }}"
         validate_certs: "{{ netapp_api_validate_certs }}"
         controller_addresses:
           - "{{ item.value.address1 }}"
           - "{{ item.value.address2 }}"
       with_dict: "{{ storage_systems }}"
       when: check_storage_system

RETURN VALUES                   *netapp_e_storage_system-return values*

   Common return values are documented here, the following are the fields unique to this module:

    Key   Returned                                                                                    Description
   msg             State of request
   string always   Sample:
                   Storage system removed.
