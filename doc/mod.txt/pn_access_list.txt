SYNOPSIS                   *pn_access_list*

     • This module can be used to create and delete an access list.

PARAMETERS                   *pn_access_list-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   pn_cliswitch                  Target switch to run the CLI on.
   string
   pn_name                       Access List Name.
   string
   pn_scope       Choices:
   -              • local        scope. Available valid values - local or fabric.
                  • fabric
   state          Choices:
   - / required   • present      State the action to perform. Use 'present' to create access-list and 'absent' to delete access-list.
                  • absent

EXAMPLES                   *pn_access_list-examples*

 - name: access list functionality
   pn_access_list:
     pn_cliswitch: "sw01"
     pn_name: "foo"
     pn_scope: "local"
     state: "present"

 - name: access list functionality
   pn_access_list:
     pn_cliswitch: "sw01"
     pn_name: "foo"
     pn_scope: "local"
     state: "absent"

 - name: access list functionality
   pn_access_list:
     pn_cliswitch: "sw01"
     pn_name: "foo"
     pn_scope: "fabric"
     state: "present"

RETURN VALUES                   *pn_access_list-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   indicates whether the CLI caused changes on the target.
   boolean
   command always   the CLI command run on the target node.
   string
   stderr  on error set of error responses from the access-list command.
   list
   stdout  always   set of responses from the access-list command.
   list
