SYNOPSIS                   *fortios_webfilter_override*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify webfilter feature and override category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_webfilter_override-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

EXAMPLES                   *fortios_webfilter_override-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure FortiGuard Web Filter administrative overrides.
     fortios_webfilter_override:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       state: "present"
       webfilter_override:
         expires: "<your_own_value>"
         id:  "4"
         initiator: "<your_own_value>"
         ip: "<your_own_value>"
         ip6: "<your_own_value>"
         new_profile: "<your_own_value> (source webfilter.profile.name)"
         old_profile: "<your_own_value> (source webfilter.profile.name)"
         scope: "user"
         status: "enable"
         user: "<your_own_value>"
         user_group: "<your_own_value> (source user.group.name)"
