SYNOPSIS                   *ce_vxlan_global*

     • Manages global attributes of VXLAN and bridge domain on HUAWEI CloudEngine devices.

PARAMETERS                   *ce_vxlan_global-parameters*

        Parameter      Choices/Defaults                                                                           Comments
   bridge_domain_id                     Specifies a bridge domain ID. The value is an integer ranging from 1 to 16777215.
   -
   nvo3_acl_extend       Choices:
   -                     • enable       Enabling or disabling the VXLAN ACL extension function.
                         • disable
   nvo3_ecmp_hash        Choices:
   -                     • enable       Load balancing of VXLAN packets through ECMP in optimized mode.
                         • disable
   nvo3_eth_trunk_hash   Choices:
   -                     • enable       Eth-Trunk from load balancing VXLAN packets in optimized mode.
                         • disable
   nvo3_gw_enhanced      Choices:
   -                     • l2           Configuring the Layer 3 VXLAN Gateway to Work in Non-loopback Mode.
                         • l3
   nvo3_prevent_loops    Choices:       Loop prevention of VXLAN traffic in non-enhanced mode. When the device works in non-enhanced mode, inter-card forwarding of VXLAN traffic may result in
   -                     • enable       loops.
                         • disable
   nvo3_service_extend   Choices:
   -                     • enable       Enabling or disabling the VXLAN service extension function.
                         • disable
   state                 Choices:
   -                     • present ←    Determines whether the config should be present or not on the device.
                         • absent
   tunnel_mode_vxlan     Choices:
   -                     • enable       Set the tunnel mode to VXLAN when configuring the VXLAN feature.
                         • disable

NOTES                   *ce_vxlan_global-notes*

   Note

     • Recommended connection is network_cli.
     • This module also works with local connections for legacy playbooks.

EXAMPLES                   *ce_vxlan_global-examples*

 - name: vxlan global module test
   hosts: ce128
   connection: local
   gather_facts: no
   vars:
     cli:
       host: "{{ inventory_hostname }}"
       port: "{{ ansible_ssh_port }}"
       username: "{{ username }}"
       password: "{{ password }}"
       transport: cli

   tasks:

   - name: Create bridge domain and set tunnel mode to VXLAN
     ce_vxlan_global:
       bridge_domain_id: 100
       nvo3_acl_extend: enable
       provider: "{{ cli }}"

RETURN VALUES                   *ce_vxlan_global-return values*

   Common return values are documented here, the following are the fields unique to this module:

      Key       Returned                                                                                  Description
   changed                 check to see if a change was made on the device
   boolean    always       Sample:
                           True
   end_state               k/v pairs of configuration after module execution
   dictionary verbose mode Sample:
                           {'bridge_domain_id': {'80': None, '90': None, '100': None}, 'nvo3_acl_extend': 'enable'}
   existing                k/v pairs of existing configuration
   dictionary verbose mode Sample:
                           {'bridge_domain': {'80': None, '90': None}, 'nvo3_acl_extend': 'disable'}
   proposed                k/v pairs of parameters passed into module

                           {'bridge_domain_id': '100', 'nvo3_acl_extend': 'enable', 'state="present"': None}
   updates                 commands sent to the device
   list       always       Sample:
                           ['bridge-domain 100', 'ip tunnel mode vxlan']
