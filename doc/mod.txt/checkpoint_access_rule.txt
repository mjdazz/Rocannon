SYNOPSIS                   *checkpoint_access_rule*

     • Manages access rules on Check Point devices including creating, updating, removing access rules objects, All operations are performed over Web Services API.

PARAMETERS                   *checkpoint_access_rule-parameters*

        Parameter       Choices/Defaults                                                                           Comments
   action               Default:         Action of the access rule (accept, drop, inform, etc).
   string               "drop"
   auto_install_policy    Choices:
   boolean                • no           Install the package policy if changes have been performed after the task completes.
                          • yes ←
   auto_publish_session   Choices:
   boolean                • no           Publish the current session if changes have been performed after task completes.
                          • yes ←
   destination                           Destination object of the access rule.
   string
   enabled                Choices:
   boolean                • no           Enabled or disabled flag.
                          • yes ←
   layer                                 Layer to attach the access rule to.
   string / required
   name                                  Name of the access rule.
   string
   policy_package       Default:         Package policy name to be installed.
   string               "standard"
   position                              Position of the access rule.
   string
   source                                Source object of the access rule.
   string
   state                Default:         State of the access rule (present or absent). Defaults to present.
   string               "present"
   targets                               Targets to install the package policy on.
   list

EXAMPLES                   *checkpoint_access_rule-examples*

 - name: Create access rule
   checkpoint_access_rule:
     layer: Network
     name: "Drop attacker"
     position: top
     source: attacker
     destination: Any
     action: Drop

 - name: Delete access rule
   checkpoint_access_rule:
     layer: Network
     name: "Drop attacker"

RETURN VALUES                   *checkpoint_access_rule-return values*

   Common return values are documented here, the following are the fields unique to this module:

             Key                             Returned                                                                            Description
   checkpoint_access_rules always, except when deleting the access rule. The checkpoint access rule object created or updated.
   list
