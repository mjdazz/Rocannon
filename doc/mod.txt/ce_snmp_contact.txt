SYNOPSIS                   *ce_snmp_contact*

     • Manages SNMP contact configurations on HUAWEI CloudEngine switches.

PARAMETERS                   *ce_snmp_contact-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   contact                       Contact information.
   - / required
   state          Choices:
   -              • present ←    Manage the state of the resource.
                  • absent

NOTES                   *ce_snmp_contact-notes*

   Note

     • Recommended connection is network_cli.
     • This module also works with local connections for legacy playbooks.

EXAMPLES                   *ce_snmp_contact-examples*

 - name: CloudEngine snmp contact test
   hosts: cloudengine
   connection: local
   gather_facts: no
   vars:
     cli:
       host: "{{ inventory_hostname }}"
       port: "{{ ansible_ssh_port }}"
       username: "{{ username }}"
       password: "{{ password }}"
       transport: cli

   tasks:

   - name: "Config SNMP contact"
     ce_snmp_contact:
       state: present
       contact: call Operator at 010-99999999
       provider: "{{ cli }}"

   - name: "Undo SNMP contact"
     ce_snmp_contact:
       state: absent
       contact: call Operator at 010-99999999
       provider: "{{ cli }}"

   Common return values are documented here, the following are the fields unique to this module:

      Key     Returned                                                                                  Description

                       True

   dictionary always   Sample:
                       {'contact': 'call Operator at 010-99999999'}
   existing   always   k/v pairs of existing aaa server
   dictionary
   proposed            k/v pairs of parameters passed into module
   dictionary always   Sample:
                       {'contact': 'call Operator at 010-99999999', 'state': 'present'}

   list       always   Sample:
                       ['snmp-agent sys-info contact call Operator at 010-99999999']
