SYNOPSIS                   *cronvar*

     • Use this module to manage crontab variables.
     • This module allows you to create, update, or delete cron variable definitions.

REQUIREMENTS                   *cronvar-requirements*

   The below requirements are needed on the host that executes this module.

     • cron

PARAMETERS                   *cronvar-parameters*

       Parameter     Choices/Defaults                                                                            Comments
   backup              Choices:
   boolean             • no ←         If set, create a backup of the crontab before it is modified. The location of the backup is returned in the backup variable by this module.
                       • yes
   cron_file                          If specified, uses this file instead of an individual user's crontab.
   string                             Without a leading /, this is assumed to be in /etc/cron.d.
                                      With a leading /, this is taken as absolute.
   insertafter                        If specified, the variable will be inserted after the variable specified.
   string                             Used with state=present.
   insertbefore                       Used with state=present. If specified, the variable will be inserted just before the variable specified.
   string
   name                               Name of the crontab variable.
   string / required
   state               Choices:
   string              • absent       Whether to ensure that the variable is present or absent.
                       • present ←
   user                               The specific user whose crontab should be modified.
   string                             This parameter defaults to root when unset.
   value                              The value to set this variable to.
   string                             Required if state=present.

EXAMPLES                   *cronvar-examples*

 - name: Ensure entry like "[email protected]" exists
   cronvar:
     name: EMAIL
     value: [email protected]

 - name: Ensure a variable does not exist. This may remove any variable named "LEGACY"
   cronvar:
     name: LEGACY
     state: absent

 - name: Add a variable to a file under /etc/cron.d
   cronvar:
     name: LOGFILE
     value: /var/log/yum-autoupdate.log
     user: root
     cron_file: ansible_yum-autoupdate
