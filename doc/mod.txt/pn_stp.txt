SYNOPSIS                   *pn_stp*

     • This module can be used to modify Spanning Tree Protocol parameters.

PARAMETERS                   *pn_stp-parameters*

          Parameter        Choices/Defaults                                                                         Comments
   pn_bpdus_bridge_ports     Choices:
   boolean                   • no           BPDU packets to bridge specific port.
                             • yes
   pn_bridge_id                             STP bridge id.
   string
   pn_bridge_priority      Default:         STP bridge priority.
   string                  "32768"
   pn_cliswitch                             Target switch to run the CLI on.
   string
   pn_enable                 Choices:
   boolean                   • no           enable or disable STP
                             • yes
   pn_forwarding_delay     Default:         STP forwarding delay between 4 and 30 secs.
   string                  "15"
   pn_hello_time           Default:         STP hello time between 1 and 10 secs.
   string                  "2"
   pn_max_age              Default:         maximum age time between 6 and 40 secs.
   string                  "20"
   pn_mst_config_name                       Name for MST Configuration Instance.
   string
   pn_mst_max_hops         Default:         maximum hop count for mstp bpdu.
   string                  "20"
   pn_root_guard_wait_time Default:         root guard wait time between 0 and 300 secs. 0 to disable wait.
   string                  "20"
   pn_stp_mode               Choices:
   string                    • rstp         STP mode.
                             • mstp
   state                     Choices:       State the action to perform. Use update to stp.
   string / required         • update

EXAMPLES                   *pn_stp-examples*

 - name: Modify stp
   pn_stp:
     pn_cliswitch: "sw01"
     state: "update"
     pn_hello_time: "3"
     pn_stp_mode: "rstp"

RETURN VALUES                   *pn_stp-return values*

   Common return values are documented here, the following are the fields unique to this module:

     Key   Returned                                                                                    Description
   changed always   indicates whether the CLI caused changes on the target.
   boolean
   command always   the CLI command run on the target node.
   string
   stderr  on error set of error responses from the stp command.
   list
   stdout  always   set of responses from the stp command.
   list
