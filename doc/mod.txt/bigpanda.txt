SYNOPSIS                   *bigpanda*

     • Notify BigPanda when deployments start and end (successfully or not). Returns a deployment object containing all the parameters for future module calls.

PARAMETERS                   *bigpanda-parameters*

     Parameter        Choices/Defaults                                                                              Comments
   component                                The name of the component being deployed. Ex: billing
   - / required                             aliases: name
   description                              Free text description of the deployment.
   -
   env                                      The environment name, typically 'production', 'staging', etc.
   -
   hosts          Default:                  Name of affected host name. Can be a list.
   -              "machine\u0027s hostname" aliases: host
   owner                                    The person responsible for the deployment.
   -
                    Choices:
   state            • started               State of the deployment.
   - / required     • finished
                    • failed
   token                                    API token.
   - / required
   url            Default:                  Base URL of the API server.
   -              "https://api.bigpanda.io"
   validate_certs   Choices:
   boolean          • no                    If no, SSL certificates for the target url will not be validated. This should only be used on personally controlled sites using self-signed certificates.
                    • yes ←
   version                                  The deployment version.
   - / required

EXAMPLES                   *bigpanda-examples*

 - bigpanda:
     component: myapp
     version: '1.3'
     token: '{{ bigpanda_token }}'
     state: started

 - bigpanda:
     component: myapp
     version: '1.3'
     token: '{{ bigpanda_token }}'
     state: finished

 # If outside servers aren't reachable from your machine, use delegate_to and override hosts:
 - bigpanda:
     component: myapp
     version: '1.3'
     token: '{{ bigpanda_token }}'
     hosts: '{{ ansible_hostname }}'
     state: started
   delegate_to: localhost
   register: deployment

 - bigpanda:
     component: '{{ deployment.component }}'
     version: '{{ deployment.version }}'
     token: '{{ deployment.token }}'
     state: finished
   delegate_to: localhost
