SYNOPSIS                   *fortios_firewall_ipv6_eh_filter*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify firewall feature and ipv6_eh_filter category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_firewall_ipv6_eh_filter-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_firewall_ipv6_eh_filter-parameters*

          Parameter        Choices/Defaults                                                                         Comments
   firewall_ipv6_eh_filter Default:         Configure IPv6 extension header filter.
   dictionary              null
         auth                Choices:
         string              • enable       Enable/disable blocking packets with the Authentication header .
                             • disable
         dest_opt            Choices:
         string              • enable       Enable/disable blocking packets with Destination Options headers .
                             • disable
         fragment            Choices:
         string              • enable       Enable/disable blocking packets with the Fragment header .
                             • disable
         hdopt_type                         Block specific Hop-by-Hop and/or Destination Option types (max. 7 types, each between 0 and 255).
         integer
         hop_opt             Choices:
         string              • enable       Enable/disable blocking packets with the Hop-by-Hop Options header .
                             • disable
         no_next             Choices:
         string              • enable       Enable/disable blocking packets with the No Next header
                             • disable
         routing             Choices:
         string              • enable       Enable/disable blocking packets with Routing headers .
                             • disable
         routing_type                       Block specific Routing header types (max. 7 types, each between 0 and 255).
         integer
   host                                     FortiOS or FortiGate IP address.
   string
   https                     Choices:
   boolean                   • no           Indicates if the requests towards FortiGate must use HTTPS protocol.
                             • yes ←
   password                Default:         FortiOS or FortiGate password.
   string                  ""
   ssl_verify                Choices:
   boolean                   • no           Ensures FortiGate certificate must be verified by a proper CA.
   added in 2.9              • yes ←
   username                                 FortiOS or FortiGate username.
   string
   vdom                    Default:         Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different unit.
   string                  "root"

NOTES                   *fortios_firewall_ipv6_eh_filter-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_firewall_ipv6_eh_filter-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure IPv6 extension header filter.
     fortios_firewall_ipv6_eh_filter:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       firewall_ipv6_eh_filter:
         auth: "enable"
         dest_opt: "enable"
         fragment: "enable"
         hdopt_type: "6"
         hop_opt: "enable"
         no_next: "enable"
         routing: "enable"
         routing_type: "10"
