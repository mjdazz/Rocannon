SYNOPSIS                   *vmware_category*

     • This module can be used to create / delete / update VMware categories.
     • Tag feature is introduced in vSphere 6 version, so this module is not supported in the earlier versions of vSphere.
     • All variables and VMware object names are case sensitive.

REQUIREMENTS                   *vmware_category-requirements*

   The below requirements are needed on the host that executes this module.

     • python >= 2.6
     • PyVmomi
     • vSphere Automation SDK

PARAMETERS                   *vmware_category-parameters*

        Parameter       Choices/Defaults                                                                           Comments
   category_cardinality   Choices:       The category cardinality.
   string                 • multiple ←   This parameter is ignored, when updating existing category.
                          • single
   category_description Default:         The category description.
   string               ""               This is required only if state is set to present.
                                         This parameter is ignored, when state is set to absent.
   category_name                         The name of category to manage.
   string / required
   hostname                              The hostname or IP address of the vSphere vCenter server.
   string                                If the value is not specified in the task, the value of environment variable VMWARE_HOST will be used instead.
   new_category_name                     The new name for an existing category.
   string                                This value is used while updating an existing category.
   password                              The password of the vSphere vCenter server.
   string                                If the value is not specified in the task, the value of environment variable VMWARE_PASSWORD will be used instead.
                                         aliases: pass, pwd
   protocol               Choices:
   string                 • http         The connection to protocol.
                          • https ←
                                         The state of category.
                          Choices:       If set to present and category does not exists, then category is created.
   state                  • present ←    If set to present and category exists, then category is updated.
   string                 • absent       If set to absent and category exists, then category is deleted.
                                         If set to absent and category does not exists, no action is taken.
                                         Process of updating category only allows name, description change.
   username                              The username of the vSphere vCenter server.
   string                                If the value is not specified in the task, the value of environment variable VMWARE_USER will be used instead.
                                         aliases: admin, user
   validate_certs         Choices:       Allows connection when SSL certificates are not valid.
   boolean                • no           Set to no when certificates are not trusted.
                          • yes ←        If the value is not specified in the task, the value of environment variable VMWARE_VALIDATE_CERTS will be used instead.

NOTES                   *vmware_category-notes*

   Note

     • Tested on vSphere 6.5

EXAMPLES                   *vmware_category-examples*

 - name: Create a category
   vmware_category:
     hostname: "{{ vcenter_server }}"
     username: "{{ vcenter_user }}"
     password: "{{ vcenter_pass }}"
     category_name: Sample_Cat_0001
     category_description: Sample Description
     category_cardinality: 'multiple'
     state: present

 - name: Rename category
   vmware_category:
     hostname: "{{ vcenter_server }}"
     username: "{{ vcenter_user }}"
     password: "{{ vcenter_pass }}"
     category_name: Sample_Category_0001
     new_category_name: Sample_Category_0002
     state: present

 - name: Update category description
   vmware_category:
     hostname: "{{ vcenter_server }}"
     username: "{{ vcenter_user }}"
     password: "{{ vcenter_pass }}"
     category_name: Sample_Category_0001
     category_description: Some fancy description
     state: present

 - name: Delete category
   vmware_category:
     hostname: "{{ vcenter_server }}"
     username: "{{ vcenter_user }}"
     password: "{{ vcenter_pass }}"
     category_name: Sample_Category_0002
     state: absent
