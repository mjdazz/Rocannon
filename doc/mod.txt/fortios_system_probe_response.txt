SYNOPSIS                   *fortios_system_probe_response*

     • This module is able to configure a FortiGate or FortiOS (FOS) device by allowing the user to set and modify system feature and probe_response category. Examples include all parameters and
       values need to be adjusted to datasources before usage. Tested with FOS v6.0.5

REQUIREMENTS                   *fortios_system_probe_response-requirements*

   The below requirements are needed on the host that executes this module.

     • fortiosapi>=0.9.8

PARAMETERS                   *fortios_system_probe_response-parameters*

         Parameter         Choices/Defaults                                                                          Comments
   host                                       FortiOS or FortiGate IP address.
   string
   https                   Choices:
   boolean                 • no               Indicates if the requests towards FortiGate must use HTTPS protocol.
                           • yes ←
   password              Default:             FortiOS or FortiGate password.
   string                ""
   ssl_verify              Choices:
   boolean                 • no               Ensures FortiGate certificate must be verified by a proper CA.
                           • yes ←
   system_probe_response Default:             Configure system probe response.
   dictionary            null
      http_probe_value                        Value to respond to the monitoring server.
      string
                           Choices:
      mode                 • none             SLA response mode.
      string               • http-probe
                           • twamp
      password                                Twamp respondor password in authentication mode
      string
      port                                    Port number to response.
      integer
      security_mode        Choices:
      string               • none             Twamp respondor security mode.
                           • authentication
      timeout                                 An inactivity timer for a twamp test session.
      integer
                           Choices:
      ttl_mode             • reinit           Mode for TWAMP packet TTL modification.
      string               • decrease
                           • retain
   username                                   FortiOS or FortiGate username.
   string
   vdom                  Default:             Virtual domain, among those defined previously. A vdom is a virtual instance of the FortiGate that can be configured and used as a different unit.
   string                "root"

NOTES                   *fortios_system_probe_response-notes*

   Note

     • Requires fortiosapi library developed by Fortinet
     • Run as a local_action in your playbook

EXAMPLES                   *fortios_system_probe_response-examples*

 - hosts: localhost
   vars:
    host: "192.168.122.40"
    username: "admin"
    password: ""
    vdom: "root"
    ssl_verify: "False"
   tasks:
   - name: Configure system probe response.
     fortios_system_probe_response:
       host:  "{{ host }}"
       username: "{{ username }}"
       password: "{{ password }}"
       vdom:  "{{ vdom }}"
       https: "False"
       system_probe_response:
         http_probe_value: "<your_own_value>"
         mode: "none"
         password: "<your_own_value>"
         port: "6"
         security_mode: "none"
         timeout: "8"
         ttl_mode: "reinit"
