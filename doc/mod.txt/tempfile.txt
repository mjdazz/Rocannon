SYNOPSIS                   *tempfile*

     • The tempfile module creates temporary files and directories. mktemp command takes different parameters on various systems, this module helps to avoid troubles related to that.
       Files/directories created by module are accessible only by creator. In case you need to make them world-accessible you need to use file module.
     • For Windows targets, use the win_tempfile module instead.

PARAMETERS                   *tempfile-parameters*

   Parameter Choices/Defaults                                                                                Comments
   path                       Location where temporary file or directory should be created.
   path                       If path is not specified, the default system temporary directory will be used.
   prefix    Default:         Prefix of file/directory name created by module.
   string    "ansible."
   state       Choices:
   string      • directory    Whether to create file or directory.
               • file ←
   suffix    Default:         Suffix of file/directory name created by module.
   string    ""

EXAMPLES                   *tempfile-examples*

 - name: create temporary build directory
   tempfile:
     state: directory
     suffix: build

 - name: create temporary file
   tempfile:
     state: file
     suffix: temp
   register: tempfile_1

 - name: use the registered var and the file module to remove the temporary file
   file:
     path: "{{ tempfile_1.path }}"
     state: absent
   when: tempfile_1.path is defined

RETURN VALUES                   *tempfile-return values*

   Common return values are documented here, the following are the fields unique to this module:

    Key   Returned                                                                                    Description
   path            Path to created file or directory
   string success  Sample:
                   /tmp/ansible.bMlvdk
