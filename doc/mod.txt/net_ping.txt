SYNOPSIS                   *net_ping*

     • Tests reachability using ping from network device to a remote destination.
     • For Windows targets, use the win_ping module instead.
     • For targets running Python, use the ping module instead.

PARAMETERS                   *net_ping-parameters*

    Parameter   Choices/Defaults                                                                               Comments
   count        Default:         Number of packets to send.
   -            5
   dest                          The IP Address or hostname (resolvable by switch) of the remote node.
   - / required
   source                        The source IP Address.
   -
   state          Choices:
   -              • absent       Determines if the expected result is success or fail.
                  • present ←
   vrf          Default:         The VRF to use for forwarding.
   -            "default"

NOTES                   *net_ping-notes*

   Note

     • For Windows targets, use the win_ping module instead.
     • For targets running Python, use the ping module instead.
     • This module is supported on ansible_network_os network platforms. See the Network Platform Options for details.

EXAMPLES                   *net_ping-examples*

 - name: Test reachability to 10.10.10.10 using default vrf
   net_ping:
     dest: 10.10.10.10

 - name: Test reachability to 10.20.20.20 using prod vrf
   net_ping:
     dest: 10.20.20.20
     vrf: prod

 - name: Test unreachability to 10.30.30.30 using default vrf
   net_ping:
     dest: 10.30.30.30
     state: absent

 - name: Test reachability to 10.40.40.40 using prod vrf and setting count and source
   net_ping:
     dest: 10.40.40.40
     source: loopback0
     vrf: prod
     count: 20

RETURN VALUES                   *net_ping-return values*

   Common return values are documented here, the following are the fields unique to this module:

       Key     Returned                                                                                  Description
   commands             Show the command sent.
   list        always   Sample:
                        ['ping vrf prod 10.40.40.40 count 20 source loopback0']
   packet_loss          Percentage of packets lost.
   string      always   Sample:
                        0%
   packets_rx           Packets successfully received.
   integer     always   Sample:
                        20
   packets_tx           Packets successfully transmitted.
   integer     always   Sample:
                        20
   rtt                  Show RTT stats.
   dictionary  always   Sample:
                        {'avg': 2, 'max': 8, 'min': 1}
