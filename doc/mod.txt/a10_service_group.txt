SYNOPSIS                   *a10_service_group*

     • Manage SLB (Server Load Balancing) service-group objects on A10 Networks devices via aXAPIv2.

Parameters¶

         Parameter                  Choices/Defaults                                                                          Comments
   client_cert                                                  PEM formatted certificate chain file to be used for SSL client authentication.
   path                                                         This file can also include the key as well, and if the key is included, client_key is not required.
   client_key                                                   PEM formatted file that contains your private key to be used for SSL client authentication.
   path                                                         If client_cert contains both the certificate and key, this option is not required.
   force                    Choices:                            If yes do not get a cached copy.
   boolean                  • no ←                              Alias thirsty has been deprecated and will be removed in 2.13.
                            • yes                               aliases: thirsty
   force_basic_auth         Choices:
   boolean                  • no ←                              Credentials specified with url_username and url_password should be passed in HTTP Header.
                            • yes
   host                                                         Hostname or IP of the A10 Networks device.
   string / required
   http_agent             Default:                              Header to identify as, generally appears in web server logs.
   string                 "ansible-httpget"
   partition                                                    set active-partition
   -
   password                                                     Password for the username account.
   string / required                                            aliases: pass, pwd
   servers                                                      A list of servers to add to the service group. Each list item should be a dictionary which specifies the server: and port:, but can
   -                                                            also optionally specify the status:. See the examples below for details.
                                                                aliases: server, member
   service_group                                                The SLB (Server Load Balancing) service-group name
   - / required                                                 aliases: service, pool, group
                            Choices:
                            • round-robin ←
                            • weighted-rr
                            • least-connection
                            • weighted-least-connection
   service_group_method     • service-least-connection          The SLB service-group load balancing method, such as round-robin or weighted-rr.
   -                        • service-weighted-least-connection aliases: method
                            • fastest-response
                            • least-request
                            • round-robin-strict
                            • src-ip-only-hash
                            • src-ip-hash
   service_group_protocol   Choices:                            The SLB service-group protocol of TCP or UDP.
   -                        • tcp ←                             aliases: proto, protocol
                            • udp
   state                    Choices:
   -                        • present ←                         If the specified service group should exists.
                            • absent
   url                                                          HTTP, HTTPS, or FTP URL in the form (http|https|ftp)://[user[:pass]]@host.domain[:port]/path
   string
   url_password                                                 The password for use in HTTP basic authentication.
   string                                                       If the url_username parameter is not specified, the url_password parameter will not be used.
   url_username                                                 The username for use in HTTP basic authentication.
   string                                                       This parameter can be used without url_password for sites that allow empty passwords
   use_proxy                Choices:
   boolean                  • no                                If no, it will not use a proxy, even if one is defined in an environment variable on the target hosts.
                            • yes ←
   username                                                     An account with administrator privileges.
   string / required                                            aliases: admin, user
   validate_certs           Choices:                            If no, SSL certificates will not be validated. This should only be used on personally controlled devices using self-signed
   boolean                  • no                                certificates.
                            • yes ←
   write_config             Choices:                            If yes, any changes will cause a write of the running configuration to non-volatile memory. This will save all configuration changes,
   boolean                  • no ←                              including those that may have been made manually or through other modules, so care should be taken when specifying yes.
                            • yes

Notes¶

   Note

     • Requires A10 Networks aXAPI 2.1.
     • When a server doesn’t exist and is added to the service-group the server will be created.
     • Requires A10 Networks aXAPI 2.1.

Examples¶

 # Create a new service-group
 - a10_service_group:
     host: a10.mydomain.com
     username: myadmin
     password: mypassword
     partition: mypartition
     service_group: sg-80-tcp
     servers:
       - server: foo1.mydomain.com
         port: 8080
       - server: foo2.mydomain.com
         port: 8080
       - server: foo3.mydomain.com
         port: 8080
       - server: foo4.mydomain.com
         port: 8080
         status: disabled
