SYNOPSIS                   *bcf_switch*

     • Create and remove a Big Cloud Fabric switch.

PARAMETERS                   *bcf_switch-parameters*

     Parameter    Choices/Defaults                                                                              Comments
   access_token                    Big Cloud Fabric access token. If this isn't set then the environment variable BIGSWITCH_ACCESS_TOKEN is used.
   -
   controller                      The controller IP address.
   - / required
   fabric_role      Choices:
   - / required     • spine        Fabric role of the switch.
                    • leaf
   leaf_group                      The leaf group of the switch if the switch is a leaf.
   -
   mac                             The MAC address of the switch.
   - / required
   name                            The name of the switch.
   - / required
   state            Choices:
   -                • present ←    Whether the switch should be present or absent.
                    • absent
   validate_certs   Choices:
   boolean          • no           If false, SSL certificates will not be validated. This should only be used on personally controlled devices using self-signed certificates.
                    • yes ←

EXAMPLES                   *bcf_switch-examples*

 - name: bcf leaf switch
   bcf_switch:
     name: Rack1Leaf1
     fabric_role: leaf
     leaf_group: R1
     mac: 00:00:00:02:00:02
     controller: '{{ inventory_hostname }}'
     state: present
     validate_certs: false
