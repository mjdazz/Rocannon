SYNOPSIS                   *ce_snmp_community*

     • Manages SNMP community configuration on HUAWEI CloudEngine switches.

PARAMETERS                   *ce_snmp_community-parameters*

       Parameter        Choices/Defaults                                                                            Comments
   access_right         Choices:
   -                    • read             Access right read or write.
                        • write
   acl_number                              Access control list number.
   -
   community_mib_view                      Mib view name.
   -
   community_name                          Unique name to identify the community.
   -
   group_name                              Unique name to identify the SNMPv3 group.
   -
   notify_view                             Mib view name for notification.
   -
   read_view                               Mib view name for read.
   -
                        Choices:
   security_level       • noAuthNoPriv     Security level indicating whether to use authentication and encryption.
   -                    • authentication
                        • privacy
   state                Choices:
   -                    • present ←        Manage the state of the resource.
                        • absent
   write_view                              Mib view name for write.
   -

NOTES                   *ce_snmp_community-notes*

   Note

     • This module requires the netconf system service be enabled on the remote device being managed.
     • Recommended connection is netconf.
     • This module also works with local connections for legacy playbooks.

EXAMPLES                   *ce_snmp_community-examples*

 - name: CloudEngine snmp community test
   hosts: cloudengine
   connection: local
   gather_facts: no
   vars:
     cli:
       host: "{{ inventory_hostname }}"
       port: "{{ ansible_ssh_port }}"
       username: "{{ username }}"
       password: "{{ password }}"
       transport: cli

   tasks:

   - name: "Config SNMP community"
     ce_snmp_community:
       state: present
       community_name: Wdz123456789
       access_right: write
       provider: "{{ cli }}"

   - name: "Undo SNMP community"
     ce_snmp_community:
       state: absent
       community_name: Wdz123456789
       access_right: write
       provider: "{{ cli }}"

   - name: "Config SNMP group"
     ce_snmp_community:
       state: present
       group_name: wdz_group
       security_level: noAuthNoPriv
       acl_number: 2000
       provider: "{{ cli }}"

   - name: "Undo SNMP group"
     ce_snmp_community:
       state: absent
       group_name: wdz_group
       security_level: noAuthNoPriv
       acl_number: 2000
       provider: "{{ cli }}"

   Common return values are documented here, the following are the fields unique to this module:

      Key     Returned                                                                                  Description
   changed             check to see if a change was made on the device

                       True
   end_state           k/v pairs of aaa params after module execution
   dictionary always   Sample:
                       {'snmp v3 group': {'snmp_group': ['wdz_group', 'noAuthNoPriv', '2000']}}
   existing   always   k/v pairs of existing aaa server
   dictionary
   proposed            k/v pairs of parameters passed into module
   dictionary always   Sample:
                       {'acl_number': '2000', 'group_name': 'wdz_group', 'security_level': 'noAuthNoPriv', 'state': 'present'}
   updates             command sent to the device
   list       always   Sample:
                       ['snmp-agent group v3 wdz_group noauthentication acl 2000']
