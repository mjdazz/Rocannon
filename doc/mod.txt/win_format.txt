SYNOPSIS                   *win_format*

     • The win_format module formats an existing volume or a new volume on an existing partition on Windows

PARAMETERS                   *win_format-parameters*

        Parameter       Choices/Defaults                                                                           Comments
   allocation_unit_size                  Specifies the cluster size to use when formatting the volume.
   integer                               If no cluster size is specified when you format a partition, defaults are selected based on the size of the partition.
   compress               Choices:       Enable compression on the resulting NTFS volume.
   boolean                • no           NTFS compression is not supported where allocation_unit_size is more than 4096.
                          • yes
   drive_letter                          Used to specify the drive letter of the volume to be formatted.
   string
                          Choices:
                          • ntfs
   file_system            • refs         Used to specify the file system to be used when formatting the target volume.
   string                 • exfat
                          • fat32
                          • fat
   force                  Choices:
   boolean                • no           Specify if formatting should be forced for volumes that are not created from new partitions or if the source and target file system are different.
                          • yes
   full                   Choices:       A full format writes to every sector of the disk, takes much longer to perform than the default (quick) format, and is not recommended on storage that is
   boolean                • no           thinly provisioned.
                          • yes          Specify true for full format.
   integrity_streams      Choices:
   boolean                • no           Enable integrity streams on the resulting ReFS volume.
                          • yes
   label                                 Used to specify the label of the volume to be formatted.
   string
   large_frs              Choices:
   boolean                • no           Specifies that large File Record System (FRS) should be used.
                          • yes
   new_label                             Used to specify the new file system label of the formatted volume.
   string
   path                                  Used to specify the path to the volume to be formatted.
   string

NOTES                   *win_format-notes*

   Note

     • One of three parameters (drive_letter, path and label) are mandatory to identify the target volume but more than one cannot be specified at the same time.
     • This module is idempotent if force is not specified and file system labels remain preserved.
     • For more information, see https://docs.microsoft.com/en-us/previous-versions/windows/desktop/stormgmt/format-msft-volume

EXAMPLES                   *win_format-examples*

 - name: Create a partition with drive letter D and size 5 GiB
   win_partition:
     drive_letter: D
     partition_size: 5 GiB
     disk_number: 1

 - name: Full format the newly created partition as NTFS and label it
   win_format:
     drive_letter: D
     file_system: NTFS
     new_label: Formatted
     full: True
